## [v0.7.3] - 2026-05-21

### 新增功能 (Added)
* 新增 Ultra clean frame v1：`ultra` payload 现在会被封装为 `preamble | sync | version | flags | payload_length | payload | crc16`，再进入 `16-FSK` symbol 编码，为后续同步、边界验证和错误检测提供稳定协议层。
* 新增 `DecodeFrameToPayload(...)` / `EncodePayloadToFrame(...)`，decode 会验证 `preamble`、`sync`、`version`、`flags`、`payload_length` 和 CRC-16/CCITT-FALSE 后只返回可信 payload。
* 新增 Ultra full-frame follow timeline，向 API 暴露每个 frame symbol 的起止 sample、nibble 值、carrier frequency、frame section 与 payload 归属，供 Android visual 展示真实 frame 音频结构。

### 技术改进/重构 (Changed/Refactor)
* 调整 Ultra PCM encode/decode 主链路，从旧的裸 payload nibble stream 改为 clean frame nibble stream；`payload_begin_sample`、`payload_sample_count` 和 `total_pcm_sample_count` 现在按 frame header / payload / CRC 的真实位置计算。
* 调整 Ultra binary group timeline，为 payload nibble 补充对应 carrier frequency，并让 payload token/timeline 起点对齐 frame 内 payload 的真实 sample 位置。
* 扩展 `bag_api` 的 follow data / encode result layout，把 Ultra frame timeline 纳入 C API 复制、生命周期和同步测试覆盖。

### 修复 (Fixed)
* 修复 Ultra decode 过去只能按“全部 symbol 都是 payload”解释的问题。现在 frame 长度和 CRC 不匹配时会拒绝输出 payload，避免把损坏或边界错误的数据当成有效文本。
