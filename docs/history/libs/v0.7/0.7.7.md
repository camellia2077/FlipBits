## [v0.7.7] - 2026-06-15

### 新增功能 (Added)
* 增加 `bag.voice.fx` / `bag_api` Voice FX 能力，向 Android 暴露 audio-to-audio 处理入口、stream processor、diagnostics 轨道和 `Machine Voice` / `Binharic` / `Signal Cant` / `Robot Vox` / `Raw Constant` / `Voice Trigger` preset。
* 增加 flash-style 非文本副轨生成能力，让 Android 可以在选择 dual track preset 后继续选择 `Standard` / `Litany` / `Hostility` / `Collapse` / `Zeal` / `Void` voicing style 作为第二轨音色，而不是只能从文本 payload 生成 flash 音频。
* 增加 single track 主轨变声处理，支撑 Android 的 `Metal`、`Code`、`Robot` 模式直接转换输入音频本身，并通过 diagnostics 区分 `main_voice`、`subvoice` 与 `signal_overlay`。

### 技术改进/重构 (Changed/Refactor)
* 拆分 Voice FX API 复制、生命周期和入口实现，保持 `bag_api.h` C ABI 对 Android JNI 稳定，同时把 core voice preset 映射集中在专用 entrypoint。
* 调整 dual track 混音边界，区分 `Raw Constant` 的连续副轨、`Voice Trigger` 的阈值触发副轨和 `Binharic` 的机械化主轨加副轨，并为 `Litany` 提供 style-specific final mix。
* 增强 `Metal`、`Code`、`Robot` 与 `Binharic` 的音色目标，加入内部 parallel mechanical layer、protocol overlay、刻板机器人声和 Binharic 输出补偿等主轨/副轨处理。

### 修复 (Fixed)
* 修复 `Signal Cant` overlay 只依赖 envelope delta 导致 diagnostics 里 `signal_overlay` 全零的问题，改为支持 level crossing 触发机器语言事件。
* 补充 Voice FX API 和 module smoke 覆盖，验证 preset、dual track 副轨、diagnostics 轨道、stream processor 和非法参数 contract。
