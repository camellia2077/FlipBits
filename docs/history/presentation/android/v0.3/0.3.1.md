## [v0.3.1] - 2026-04-22

### 新增功能 (Added)
* 新增 Android `双色主题` 风格层，和原有 `Material` palette 体系分层共存；`Config` 页现在支持在 `Material / Dual-tone` 两种主题风格间切换，`Dual-tone` 下提供按原创风格命名的双色主题组，并明确标注为固定风格、不跟随系统明暗模式。
* 新增 Audio 页机械仪式风格的常驻编码图标；输入卡片在空闲态也会显示完整 icon，编码时外层与内层齿轮按先后次序缓慢启动，中心虚线环按分段进度显示，保留原有 phase 文案、线性进度条和按钮百分比。
* 新增 Audio 页长文本输入辅助：输入区支持两行预览、`TXT` 全屏编辑入口、完整内容弱提示，以及 `短 / 长` 示例长度切换；切到 `长` 时会从新的四组记录体/编年体长文本池中随机抽样。
* 新增 Android 俄语、西班牙语、葡萄牙语与德语界面文案资源，并补齐 `Config`、`Dual-tone`、`flash voicing`、`saved audio`、`decoded text`、`library` 等核心术语的多语言口径。

### 技术改进/重构 (Changed/Refactor)
* 重构 Android 主题体系，将原先单层的 `palette + Theme mode` 扩展为“主题风格 + 风格内配置”两层结构；`Material` 风格继续保留 palette 与 Theme mode，`Dual-tone` 风格则固定走项目内的双色主题映射，不再把双色方案混入普通 Material palette 心智。
* 重构 Android 表现层的双色用色边界，新增面向风格的 UI color token 与编码图标配色映射，让展开箭头、已选标签、选中描边、输入区标题、底部栏选中/未选中图标与编码 icon 都能按浅底实体 accent、深底实体 accent、深底能量 accent 的不同策略落色。
* 重构 `Config` 页与 App 壳层职责，把主题外壳、底部栏、主 scaffold、主题映射、语言区、主题外观区与通用选择组件拆到独立文件，并将 `PaletteCatalog` 轻量拆为按色系分组的 catalog 文件，降低 `ConfigTabScreen` 与 `AudioAndroidAppShell` 的职责混杂。
* 重构 Audio 页输入与结果区的交互组织：输入卡片、`flash voicing` 区与部分设置卡片现在支持独立折叠/展开，设置页中的语言与配色分组也支持持久化记忆展开状态，整体更贴近“常用控制收口、需要时再展开”的音乐应用式布局。
* 新增并对齐 Android 设计说明文档 `docs/design/android/android-dual-tone-theme.md` 与 `docs/design/android/android-encode-glyph-colors.md`，把双色主题排序、icon 在浅底/深底下的映射规则、以及底部导航为何保留独立策略等意图落成项目内真源。

### 修复 (Fixed)
* 修复 `Dual-tone` 主题下多个高频交互元素对比度不足的问题；`Config` 页展开箭头、关于页箭头、`已选/Select` 标签、选中描边、Audio 页下拉箭头、`flash voicing` 选择框与输入/解析动作按钮的选中态现在统一使用更高对比的容器、描边和标签策略。
* 修复浅底双色主题下底部导航栏与迷你播放栏在未选中态、选中态和背景层之间容易融色的问题；底栏现在始终保持更紧凑的高度，浅底双色下未选中图标改用相对亮侧颜色的原色，迷你播放栏背景也收敛到与底栏更一致的承托层。
* 修复编码图标中心虚线环的语义和加载顺序偏差；中心环现改为围绕 `12` 点方向位翻转概念的双侧分段逻辑，工作态从首段右半开始，再按相对方位顺序点亮对应半段，避免此前一整圈密集短线和左右顺序反转的问题。
* 修复深色双色方案中部分主题的强调色过近黑、近灰导致的可读性问题；`黑红仪轨`、`远古王朝` 等主题现改用更可辨识的深红与收敛后的绿/青 accent，并同步提升设置页与导航区域的识别度。
