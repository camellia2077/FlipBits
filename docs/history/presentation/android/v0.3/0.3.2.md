## [v0.3.2] - 2026-04-23

### 新增功能 (Added)
* 新增 Android 播放跟随解析视图：生成或选中音频后，播放器详情现在支持直接查看 payload 的 `hex` / `bin` 解析结果，并可随着可视化波形播放同步高亮当前对应的 `hex/bin` 内容，帮助用户把“听到的节奏”和“看到的编码内容”对上。
* 新增面向迷你播放器的文本上下文反馈：`mini player` 与播放器详情现在会围绕当前播放音频展示最近解析到的 token 上下文，让用户无论停留在 `Visual` 还是 `Lyrics`，都能快速知道当前正在跟随的文本位置。
* 新增播放器详情中的按需解析触发：打开播放器详情或首次切到 `Lyrics` 时，会针对当前 `mini player` 选中的音频触发 decode；最新生成音频继续直接携带 follow 数据，已保存旧音频则会在第一次查看时自动补齐 follow，避免只能看到旧解析结果或完全没有歌词跟随。

### 技术改进/重构 (Changed/Refactor)
* 重构 Audio 页顶部入口，把原先单独的 `Audio` 标题收敛为正式的 `AudioModeSwitcherBar (flash/pro/ultra)`，让模式切换直接成为页面主入口，并为后续不同传输模式的独立表达预留稳定结构。
* 重构播放器详情与迷你播放器的内容层级：`Visual` 区中的核心内容整体下移，更贴近用户先看主内容、再看控制区的使用路径；迷你播放器也同步补齐更稳定的覆盖层结构，避免再出现被背景颜色包裹却仍然压在内容层上的违和感。
* 重构 `Config` 页的语言切换交互，语言列表改为点击展开按钮后从下方弹出，不再在卡片内部直接铺开多行选项，减少语言项过多时对 `Config` 页面可用空间的持续占用。
* 重构输入与解析动作分组，`Clear input` 不再挂在 `AudioResultCard` 的解析结果区，而是迁回 `AudioInputActionsCard` 的输入工具栏，与 `TXT` 编辑和随机示例放在同一组；`Decoded content` 底部因此只保留 `Decode` 主动作，输入侧与解析侧的职责更清晰。
* 重构双色主题的配色搭配，进一步拉开主辅色之间的视觉差异，让不同双色主题在内容区、播放区和设置区里看起来更容易区分，而不是过于接近的一组近色变化。

### 修复 (Fixed)
* 修复文本解析跟随错误绑定旧音频的问题；follow 数据现在跟随当前 `mini player` 实际选中的播放项，而不是继续沿用“解析前刚生成的音频”，避免切到已保存音频后 `Lyrics` 或 token 上下文仍显示旧内容。
* 修复 `mini player` 背景层级异常的问题，收敛其背景承托与内容覆盖关系，避免组件看起来既有独立背景色包裹、又像直接压在正文内容之上的双重层级冲突。
