## [v0.3.7] - 2026-04-28

### 新增功能 (Added)
* 新增 mini player 播放速度控制，支持通过快捷切换与滑杆调节 `0.1x` 到 `4.0x` 的播放倍速，并在播放中的生成音频与已保存音频间保持一致的速度控制入口。
* 新增超长输入自动分段生成能力：当单次 payload 超过上限时，Android 侧会先按 UTF-8 字节预算切段、逐段生成并合并 follow/decoded 结果，使长文本不再因为单段容量限制而直接失败。
* 新增自定义 dual-tone 主题预设编辑能力，可在设置页创建、编辑、删除与随机生成自定义配色，并把预设持久化到本地设置中复用。

### 技术改进/重构 (Changed/Refactor)
* 重构 Lyrics / token 跟随显示逻辑：英文等按词滚动的行继续保留单行横向跟随；中日韩等密集字符行改为优先按歌词原文自动换行；token 卡片在可拆分时升级为“文本 / HEX / 字符轨”三层结构，并对单字 CJK token 自动跳过第三行字符轨，减少密集歌词被右侧遮挡或信息重复的问题。
* 调整 dual-tone 视觉 token 体系，把 mini player 图标、follow token 卡片、设置分组、输入框和波形辅助色从 `Material 3` 默认色槽中解耦，统一走应用级视觉 token，让品牌主题下的图标、边框和强调色层次更稳定。
* 重构样例文本与多语言资源装载链路，抽出 `AndroidSampleCatalog`、拆分语言字符串职责文件，并同步更新 Android 本地化与播放器设计文档，为后续语言质量维护提供更稳定的资源入口。

### 修复 (Fixed)
* 修复 follow token 设计层与底层 tokenization 不一致的问题，统一把标点作为独立 token 参与 Lyrics / token tape / HEX-BIN 高亮，避免 `word+punctuation` 黏连导致的卡片显示与播放跟随错位。
* 修复多语言界面与样例文案的翻译质量问题，重点优化 `German`、`Spanish`、`French`、`Italian`、`Japanese`、`Korean`、`Chinese` 等资源中的术语、语气与 XML 文本细节，使设置页、状态文案与样例文本更自然一致。
* 修复保存音频元数据在 Android 侧的透传与展示缺失，已保存条目现在能够读出生成时间、输入来源、采样率与 payload 字节数，长文本分段生成后的文件也能保留对应 metadata 信息。
