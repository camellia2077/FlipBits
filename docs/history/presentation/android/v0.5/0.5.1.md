## [v0.5.1] - 2026-05-21

### 新增功能 (Added)
* 新增 Mini Morse 输入翻译辅助：在 `mini` 模式输入或粘贴 Morse notation 时，输入区会直接显示对应文本，支持常见点划符号、空白分词和 `/`、`|` 单词分隔。
* 新增 transport mode 持久化：首次安装仍默认 `flash`，用户切换到 `mini`、`pro` 或 `ultra` 后会保存当前模式，重新进入应用时恢复上次选择。
* 新增 Ultra full-frame visual 接入：Ultra visual 现在可以使用 native 返回的 frame symbol timeline 展示 preamble、sync、header、payload 和 CRC 对应的真实 `16-FSK` symbol。

### 技术改进/重构 (Changed/Refactor)
* 调整 Mini speed preset，从 `Slow` / `Standard` / `Fast` 改为明确的 `10 WPM` / `15 WPM` / `20 WPM`，默认档为 `15 WPM`，并同步 metadata style 映射、自动化参数、多语言文案和测试。
* 调整 Audio 页面输入区和结果区的视觉分组，使用轻量 `AudioSectionContainer` 包裹主要卡片；dual-tone 主题的 group container 会在 `6%`、`10%`、`14%` 混色间选择，保证和背景有最小视觉差。
* 调整输入框提示文案位置，把 `Longer passages stay collapsed here ...` 类提示移动到输入框下方，避免和 action/status 区混在一起。
* 调整 Ultra visual 布局，回收 detail 页面下方未使用空间给歌词/visual 预览，并保留 debug 版 `UltraVisualLayout` 尺寸日志入口用于实机排查。
* 扩展 Android native bridge 和 domain DTO，传递 Ultra frame section、nibble、carrier frequency 与 payload 归属信息，让 UI 不再从 payload group 猜 visual frame 结构。
* 更新 Android 版本号为 `0.5.1`。

### 修复 (Fixed)
* 修复 Ultra visual 过去只按 payload binary group 展示的问题。现在 visual 跟随 full-frame symbol timeline，而 Binary/Hex/Text token 高亮仍只对齐 payload 正文范围。
* 修复 Mini speed id 和 transport mode id 读取时对大小写/空白不够宽容的问题，持久化恢复会先做 trim/lowercase 再匹配。
