## [v0.5.3] - 2026-05-25

### 新增功能 (Added)
* 新增 Android native package 对 `bag.flash.phy_rules`、`bag.transport.encode_operation_steps` 和 `bag.transport.encode_work_plan` 的 package-owned 声明与 source owner，release/staging/debug 都能继续在非 module lane 下消费最新 `libs` 生成流程边界。

### 技术改进/重构 (Changed/Refactor)
* 重构 JNI follow hydration，把 follow entry 构造、hydration 和 marshalling 从 `jni_bridge_helpers.cpp` 拆到独立 C++ 文件，降低终态 PCM、follow data 和 UI view data 之间的桥接耦合。
* 重构 Flash visual state，把原来的 `FlashSignalVisualizationState` 拆成 analysis、bucket source keys、playback readout、timeline strategy 和 render state 等更小边界，让可视化状态、播放读数和渲染策略可以独立演进。
* 调整 Android native package 接线，跟随 `libs` 的 Flash `phy_rules`、transport encode operation steps/work-plan 和 WAV metadata parse 边界更新 `CMakeLists.txt` 与 package-private include。
* 更新 Android presentation 版本号为 `0.5.3`。

### 修复 (Fixed)
* 修复 `assembleRelease` 在 Android native package 编译阶段因 `audio_core` / `bag_api` 内部 include 顺序不满足 release package-owned source 编译而失败的问题，release APK 可以重新生成。
* 修复 Android quality 检查中的 ktlint 格式问题，包括 import 顺序、缩进和类尾多余空行。
