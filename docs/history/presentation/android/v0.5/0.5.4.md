## [v0.5.4] - 2026-05-26

### 新增功能 (Added)
* 新增 Custom `dual-tone` / Material 颜色配置的批量纯文本导入导出边界：Material 导出统一使用 `name=...`、`primary=...` 两行，`dual-tone` 导出统一使用 `name=...` 单独一行与 `primary=... secondary=... outline=...` 第二行；同时支持按分享代码顺序还原多组颜色，并让新建或导入的颜色统一从列表最上方插入。
* 新增 `The Sacred Machine` 内置 `dual-tone` 主题 `Xeno Code`，并补齐多语言主题标题、描述和无障碍文案。
* 新增 mini player detail 歌词拖动选行交互：支持上下拖动进入选行态、显示固定选行横线与目标时间、松手后按当前歌词行 seek。

### 技术改进/重构 (Changed/Refactor)
* 重构 Custom 颜色重复判定与导入匹配规则：`dual-tone` custom 只有 `name`、`primary`、`secondary`、`outline` 全部一致才视为重复，Material custom 只有 `name` 与 `primary` 一致才视为重复。
* 调整 `dual-tone` 颜色复制导出逻辑，内置主题导出的 `name` 现在优先使用当前语言 `XML` 里的本地化标题，不再回退到英文常量名或带下划线的内部 `id`；自定义 `dual-tone` 仍保留用户输入名称。
* 调整内置 `dual-tone` 分组 section 的 header action，仅在展开态显示复制入口；折叠态隐藏复制 action，降低未展开分组的操作噪音。
* 调整 Custom `dual-tone` 编辑页的输入框描边语义，在 `dual-tone` 模式下统一走 `outlineColor`，不再沿用 `secondaryColor`。
* 调整 `dual-tone` 预览与标题文字语义：custom 预览卡片中的字母 `A` 现在改为使用 `outlineColor`；内置 `dual-tone` 分组标题与 `Read result` 中 `Text` 标题改为直接使用 `onSurface`，不再跟随混色后的次级文字链路。
* 重构 expanded player / full Lyrics 的歌词交互边界：detail 恢复播放跟随歌词样式，full Lyrics 保留独立滚动与播放控制，并在进入 full Lyrics 后隐藏 detail 顶部折叠、分享、下载图标。
* 调整 detail / full Lyrics 的选行视觉反馈，降低选行横线透明度、将选行时间移动到横线右侧并缩短横线长度，拖动选行时保持字号、行高与字重稳定，避免布局跳动。
* 调整 expanded player 手势约束，禁用垂直拖拽收起以避免与歌词上下滑动选行冲突，保留左上折叠和系统 Back 退出 detail。
* 调整 `dual-tone` 次级文字取色策略，从固定混色改成 `outlineColor first + contrast fallback to onBackground`，让主题描述和辅助文字优先保留 `outline` 语义，但在低对比配色下仍自动回退到可读文本色。
* 调整 custom `dual-tone` / Material 的删除确认弹窗配色，`Edit custom dual-tone` 与 `Edit colors` 的删除页面现在统一复用当前 preset 的 `primary` 作为弹窗背景，和编辑页主配色保持一致。
* 调整 custom `dual-tone` / Material 的删除确认弹窗背景回退策略，即使编辑中的 `primary` 输入暂时无效，`Edit custom dual-tone` 与 `Edit colors` 的删除页面也会继续使用当前 preset 的 `primary`，不再退回透明或通用弹窗底色。
* 更新 Android presentation 版本号为 `0.5.4`。

### 修复 (Fixed)
* 修复 Custom Material / `dual-tone` 颜色新增与导入时默认追加到列表底部的问题，统一改为顶部插入，保持最近新增内容可见。
* 修复多组 Custom 颜色批量导入时因 repeated head insertion 导致顺序反转的问题，导入后的顶部顺序现在与分享代码里的导出顺序一致。
* 修复切换内置 `dual-tone` 主题导致 sample flavor 变化时，sample 文本长度被强制回退到 `Short` 的问题；现在优先继承当前 session 的 `sampleShuffleState.length`，只有没有历史长度时才回退 `Short`。
* 修复冷启动时 `input text` 的 `short` / `long` 恢复不一致问题；sample 初始化与重新开启 auto-fill 现在都会显式使用已持久化的 `selectedSampleInputLength`。
* 修复 `Hazard Warning`（原 `Scarlet Guard`）内置主题名称与颜色说明长期不匹配的问题，文案现在按浅工业灰、警示橙红和深石墨描边重新描述并同步到全部本地化资源。
* 修复 release 构建下应用冷启动立即闪退的问题。根因是 `BrandThemeCatalog.kt` 顶层 `dual-tone` 文本色辅助常量的初始化顺序与启动期首屏主题构建冲突，导致 `BrandThemeCatalogKt.<clinit>` 在 Compose attach 阶段触发 `ExceptionInInitializerError` / `NullPointerException`；现在已收紧初始化顺序并改用稳定的 fallback 遍历逻辑。
