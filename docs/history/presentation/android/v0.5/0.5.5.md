## [v0.5.5] - 2026-05-30

### 新增功能 (Added)
* 增加保存音频时的 playback details 选项，允许用户控制是否把 visual/tokens 回放所需数据保存到 app 的用户存储中；重新打开本地保存音频时可用这些数据恢复阅读、visual 和 token 展示。
* 增加 Lyrics full-screen 独立全文阅读模型，生成完成后缓存完整导航文本，不再依赖会随播放窗口替换的 follow data，修复 Litany 等长内容只显示局部文本的问题。
* 增加 full-screen lyrics 阅读行列表、滑动选行 seek 和字符级播放高亮；全文会按标点和当前宽度切成视觉行，拖动时显示选取横线与时间，松手后跳到该视觉行第一个可 seek 字符。

### 技术改进/重构 (Changed/Refactor)
* 接入 `libs` 新增的 decode operation contract，把保存音频后的长音频 decode 拆成可汇报进度的阶段，避免一次性 decode 造成进度卡住或内存压力。
* 更新生成音频进度阶段文案，明确区分 `Preparing input`、`Generating audio`、`Shaping audio` 和 `Building playback data`，让输入转换、PCM 渲染、Flash voicing 后处理与播放跟随数据构建的状态更清晰。
* 重构 saved decode cache、saved playback selection、segmented decode progress 和 JNI gateway 边界，统一判断本地保存音频的 decoded follow data 是否过期，并让 UI 不再直接调用 JNI file segment decode。
* 调整 mini player 中 generated audio 标题/副标题颜色，使文字根据 mini player 背景自动取反，而不是固定使用 outline 色。
* 调整播放详情页分享按钮行为，分享生成音频时不再顺带把音频保存到本地。

### 修复 (Fixed)
* 修复长音频保存到本地后再次打开时，decode 后 visual/tokens 为空的问题。
* 修复长音频 decode 过程中进度条卡死或缺少进度反馈的问题。
* 修复长音频 decode 崩溃风险，并通过 file-backed segment decode 降低长 PCM 数据跨 JNI/VM 边界的压力。
* 修复播放中切换到其他倍速后再切回 `1.0x`，底层 `AudioTrack` 仍沿用旧倍速的问题。
