## [v0.6.0] - 2026-06-15

### 新增功能 (Added)
* 增加 Android `Voice` 页面，提供 `Clip` 和 `Live` 两种 audio-to-audio 工作流，支持录音、导入音频、处理、试听、分享和导出处理结果。
* 增加 `Single track` / `Dual track` preset 分类，`Single track` 用于转换输入人声本身，`Dual track` 用于生成主轨和第二轨后混合输出。
* 增加 `Metal`、`Code`、`Robot`、`Binharic`、`Voice Trigger`、`Raw Constant` preset 选择，并在 dual track 下提供 `Standard` / `Litany` / `Hostility` / `Collapse` / `Zeal` / `Void` voicing style 选择。
* 增加 Voice diagnostics 试听轨道，让调试时可以分别播放输入、最终输出、主轨、第二轨或 signal overlay。

### 技术改进/重构 (Changed/Refactor)
* 将原有 Audio 页面拆分为 `Data` 与 `Voice` 入口，底部导航和 ViewModel 接入独立的 Voice session state、actions、screen component 与 native gateway。
* 将 Voice 首次进入默认设为 `Dual track` / `Binharic`，并保留用户后续选择的持久化恢复逻辑，避免覆盖已保存的 track mode。
* 接入 Android native Voice FX JNI、录音、音频导入、live routing、preview player 和保存音频分享链路，使 Clip 与 Live 共用同一套 preset / voicing 配置。
* 扩展 Android 多语言字符串、测试和文档入口，覆盖 Voice 页面状态、preset 切换、dual track voicing、live preview 和导出/分享流程。

### 修复 (Fixed)
* 修复 Voice preview 与全局 playback、离开 Voice tab、保存音频选择之间的播放状态冲突，避免多个音频源同时播放。
* 修复 live preview 在不同 preset 和 dual track voicing 下的配置传递，确保 native 层收到当前选择的 preset 与 subvoice style。
