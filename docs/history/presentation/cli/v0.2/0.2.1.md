## [v0.2.1] - 2026-04-22

### 新增功能 (Added)
* 新增 CLI `licenses` 子命令；`FlipBits licenses` 现在会输出当前 CLI 第三方 notices 的首轮覆盖摘要，并指向仓库中的稳定 notices 文档 `docs/legal/cli_third_party_notices.md`，让构建、运行时与测试依赖的覆盖范围有了可直接查询的入口。
* 新增 CLI 编码进度显示；`encode` 现会在终端环境下通过 `stderr` 输出带 phase 文案的进度条，并随共享 `bag_api` encode progress 更新百分比，帮助长文本编码时感知当前阶段与完成度，同时不污染脚本可消费的 `stdout` 结果。

### 技术改进/重构 (Changed/Refactor)
* 重构 Rust CLI 命令面与帮助文案，正式收口为 `version / licenses / encode / decode` 四个主命令；顶层 `--help`、子命令 `about/long_about/after_help` 与示例文案现统一围绕“检查构建、查看 notices、做 WAV 往返”这一条主线组织。
* 重构 CLI 对 `audio_io` metadata 的消费边界，配合共享库命名收口改为围绕 `FlipBits metadata` 工作；encode 写入的 metadata、decode 读取后的错误文案、以及版本输出中的 app/core 口径现统一避免继续暴露旧品牌前缀。
* 重构 CLI 表现层的仓库接线与文档入口，补充 `apps/audio_cli/AGENTS.md` 并把命令面、进度输出、FFI 边界、构建与测试入口收成可直接执行的维护规则，降低后续继续扩展子命令和 presentation 逻辑时的入口查找成本。

### 修复 (Fixed)
* 修复 CLI 缺少稳定第三方 notice 查询入口的问题；现在无需再靠 `Cargo.toml`、锁文件或零散文档手工推断 CLI 的依赖覆盖范围。
* 修复编码进度与最终结果共用同一输出通道、容易破坏脚本消费 `stdout` 的问题；进度条现在仅在 `stderr` 且检测到终端时启用，最终 `Output WAV` 和 decode 文本输出仍保持稳定。
* 修复 decode 相关帮助与测试覆盖对当前真实命令面的表达不足；帮助信息、集成测试与错误路径校验现同步覆盖 `licenses`、自动 metadata 解码、缺失输入与截断 WAV 等关键 CLI 行为。
