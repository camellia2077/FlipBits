## [v0.2.7] - 2026-05-16

### 技术改进/重构 (Changed/Refactor)
* 调整 Rust CLI 写出的 `WBAG metadata` 版本到 `v7`，并同步更新 `apps/audio_cli/AGENTS.md` 的 `audio_io` 边界说明。这样 CLI 与当前 `libs/audio_io` / Android 保存音频链路使用的是同一代 metadata 契约，不再停留在旧版本口径。
* 调整 CLI 生成 WAV metadata 的 style 写入逻辑：`flash` 继续写入 `flash_voicing_style`，`mini` 现在也会根据 `frame_samples` 推导并写入 `mini_speed_style`。这样后续 Android 或其他消费方在读取 CLI 生成的 WAV 时，也能直接知道 `mini` 音频是 `slow / standard / fast` 哪种 speed style。
* 调整 CLI 对 `flash` style 的公开命名口径，把此前代码/参数层混用的 `Hostile` 统一收口为 `Hostility`，与 Android 设置文案、`libs` 公开枚举、WAV metadata 和 release history 保持一致，不再继续保留两套近义命名。

### 修复 (Fixed)
* 修复 Rust CLI 导出的 `mini` WAV 缺少 speed style metadata、导致下游只能从文件名或外部上下文猜测速率的问题。现在 CLI 会把 `mini_speed_style` 一起编码进 `WBAG metadata v7`，并通过回读测试保证 `mini` style 可以稳定 roundtrip。
