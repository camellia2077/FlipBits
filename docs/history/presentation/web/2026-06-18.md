## [v0.3.0] - 2026-06-18

### 新增功能 (Added)
* 新增 "Voice FX" (音频变声) 模块，支持通过 WebAssembly 桥接复用底层变声算法，实现音频生成音频 (audio-to-audio) 功能。
* 新增网页麦克风录音功能，支持通过 `MediaRecorder` 录制语音并实时生成临时 WebM 音频 Blob 传入编码器。
* 新增本地音频文件上传支持，用户可以直接导入本地音频文件进行变声处理。
* 新增 Voice FX 单轨与双轨处理模式，支持在双轨模式下自主选择细分的 Voicing Style 样式。
* 新增客户端音频前置解码与重采样，自动将多声道或异轨音频转为 16-bit Mono PCM 48kHz，对齐 Android 端离线 Voice FX 变声的统一规范。
* 新增 Web 侧性能监控工具 `perf-diagnostics.js`，统计解码、重采样、Wasm 运行与 WAV 生成等关键操作耗时并通过控制台输出。
* 新增 Voice FX 处理结果摘要（Result Summary）和专属 WAV 预览与下载控件。

### 技术改进/重构 (Changed/Refactor)
* 优化 UI 页面布局，引入双通道工作流切换页签（Workflow Tabs: Data Mode / Voice FX），提升操作聚焦度。
* 重构 Wasm 工作线程，在 `encode-worker.js` 中新增 `voice-fx` 消息处理分支，使变声处理在后台异步运行，防止界面卡顿。
* 优化 Voice FX 状态指示器，根据变声跨端规范去除虚假的百分比进度，仅按节点状态（等待、解码中、处理中、成功/失败）进行流转。
* 补齐多国语言国际化文案，为各语言包（如 `zh-CN`, `en`, `ja` 等）补充 Voice FX、录制以及相应的状态反馈文案。
* 优化 Web 侧数据编码与渲染性能，引入了以下三项关键优化以解决长文本 flash 音频生成的耗时瓶颈：
  * **主线程与 Worker 通信节流**：将 Web worker 编码 pump 预算从 2048 提高到 65536 work units，将 progress 通信限制为约 50ms 一次，UI 进度渲染限制为 100ms 一次。使 `long-flash-litany` 进度事件数从 17.2 万次降低至约 1.6 千次，UI 渲染时间开销显著降低，生成耗时缩短约 8.9s。
  * **底库 Pump 批处理大小提升**：将 C++ 内核 `Pump` 的单次渲染/后处理批次大小从 256 提升至 2048 (`kRenderPumpWorkBudget`)，提高吞吐量，使 `long-flash-litany` 耗时进一步缩短约 22.4s。
  * **控制分支按需跳过**：对于被识别为最耗时的 `payload` 阶段，通过在底库中缓存布尔状态，静态跳过当前 style traits 未启用的算法层（如 `use_metallic_layer_`、`use_collapse_tremor_`、`use_chant_swell_`、`use_payload_softclip_`）及早退边界点击计算。使 `long-flash-litany` 耗时再降低约 3.3s。
* 补充优化前后的 Web 自动化性能测试对比（使用 `python tools/run.py web perf-data` 测得的基准数据）：
  * `short-flash-standard` (短 Flash)：1440.4 ms -> 1002.8 ms (缩短 437.6 ms)
  * `long-flash-litany` (长 Flash 经多层优化后)：115955.5 ms -> 52564.4 ms (总体缩短约 63.3s，降幅达 54.6%)
  * `long-ultra` (长 Ultra 传输)：484.2 ms -> 283.8 ms (缩短 200.4 ms)
