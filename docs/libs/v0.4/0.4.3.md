## [v0.4.3] - 2026-04-23

### 新增功能 (Added)
* 新增 `libs/audio_core` 与 `libs/audio_api` 的 payload follow 能力：decode / encode 结果现在可同时返回原始 payload 的 `hex` / `binary` 表达、按字节和按 binary group 切分的 sample timeline、以及文本 token / 歌词行 / raw segment 对应的 follow 数据，作为 Android 侧“解析为 hex/bin”与“随波形播放同步高亮对应 hex/bin 字符”的底层 contract。
* 新增 `bag_api` 的正式 follow 数据边界：`bag_decode_result`、`bag_encode_result`、`bag_payload_follow_data`、`bag_text_follow_data` 现统一暴露 `raw_bytes_hex`、`raw_bits_binary`、byte timeline、binary group timeline、token timeline、lyric line timeline 与 line/token range 等结果，让多前端不必各自重建 payload 到文本/二进制的映射关系。

### 技术改进/重构 (Changed/Refactor)
* 重构 `audio_core` 的 transport follow 生成链路，新增 `bag.transport.follow` 模块并把 `BuildPayloadFollowData(...)` / `BuildTextFollowData(...)` 下沉为共享能力，让 `flash`、`pro`、`ultra` 的 encode / decode 结果走同一套 payload-to-follow 生成路径，减少不同传输模式继续各自分叉 follow 语义的风险。
* 重构 `audio_core` / `audio_api` 的结果类型边界，把 `PayloadFollowData`、`TextFollowData`、`DecodeResult` 和对应 C ABI buffer/status 字段统一收口到公共类型中，明确区分文本 decode 状态、原始 payload 可用性、hex/bin 内容状态与 follow timeline 可用性，方便前端按能力渐进显示而不是把“能解文本”和“能展示 payload 细节”绑成一个布尔语义。

### 修复 (Fixed)
* 补齐 `libs/audio_api/tests/api_sync_tests.cpp` 与 shared test support 对 follow timeline 的连续性回归，锁定 binary group timeline 从 `payload_begin_sample` 起连续覆盖到 `payload_sample_count`、且不越过 `total_pcm_sample_count` 的契约，避免后续调整 render 或 payload 切分后，Android 侧波形高亮与 hex/bin 跟随再次漂移。
