## [v0.6.0] - 2026-04-30

### 新增功能 (Added)
* 新增 `mini` 传输模式作为 Android / CLI 的 Morse code 底层 contract：`libs/audio_core` / `libs/audio_api` 现在支持 `BAG_TRANSPORT_MINI`，可对 `A-Z`、`0-9`、空格和常见 Morse 标点进行规范化编码，decode 后返回规范化文本，并通过 `BAG_VALIDATION_MINI_MORSE_ONLY` 为 Android 输入错误提示和 CLI 参数校验提供明确原因。
* 新增 `mini` Morse follow 数据，支撑 Android 的 Morse timeline visual 与 lyrics/follow 适配：encode / decode 结果现在可为 Morse payload 发布按字符和按 dot/dash tone element 切分的 sample timeline，让前端可以按原始/规范化文本或 Morse 点划节奏做播放跟随，而不必在 Kotlin 或 CLI 侧重新推导 Morse 时间轴。
* 新增 Flash 四情绪风格的正式库层取值，支撑 Android 风格选择器和 CLI 风格参数统一使用 `Steady`、`Hostile`、`Litany`、`Collapse`：旧 `coded_burst` / `ritual_chant` / `deep_ritual` 语义被移出公开 contract，四个新风格分别接入 C API、core config、metadata 枚举、roundtrip 与 validation 测试。
* 新增 Flash 可跳过 silence 的变长 payload profile，为 Android 侧 `Litany` / `Collapse` 的波形、歌词跟随和可解码音效提供底层能力：`Litany` 支持吟诵式小停顿、空格停顿和标点大停顿，`Collapse` 支持确定性不规则 hesitation / stutter silence，同时 decode 通过 silence skipping 保持文本可还原。

### 技术改进/重构 (Changed/Refactor)
* 重构 transport mode 枚举顺序为 `mini / flash / pro / ultra`，并同步 `bag_transport_mode_name(...)`、`bag_try_parse_transport_mode(...)`、core transport facade、structured encode/decode 和 async API 测试，目的是让 Android、CLI 与库层共享同一套产品模式顺序和命令行拼写。
* 重构 Flash signal profile 与 voicing flavor 语义，删除旧 `CodedBurst / RitualChant / DeepRitual` 公开口径，改为 `Steady / Hostile / Litany / Collapse`，并把频率、payload timing、preamble / epilogue shell 与 voicing texture 分别收口到对应风格配置中，避免 Android 和 CLI 在各自 UI/参数层维护一套重复映射。
* 调整 Flash 风格频率与音色底层参数，为 Android 实时预览、保存音频和 CLI encode 输出提供一致听感：`Steady` 使用 `300 / 600 Hz`，`Litany` 使用 `220 / 440 Hz` 并叠加低频 drone、mechanical throat 与 chant swell，`Hostile` 使用 `450 / 900 Hz`，`Collapse` 使用 `280 / 560 Hz`。
* 重构 Flash payload layout，使 payload timeline 能同时表达 voiced bit chunk 与 silence chunk；follow byte span 会包含字节后的 cadence silence，但 binary group timeline 只发布有声 low/high bit，目的是让 Android 波形和 lyrics/follow 能展示真实停顿，同时不会把停顿误当成数据位。
* 更新 `WBAG` metadata mode / flash style 枚举，让 Android 保存音频库和 CLI WAV metadata 能记录 `mini` 模式与四个 Flash 情绪值；旧 Flash style 命名不再作为新 contract 暴露，metadata roundtrip 测试覆盖所有新 mode/style 组合。
* 扩展 C++20 module wiring，为 `bag.mini.codec`、`bag.mini.phy_clean` 和 transport follow/facade 暴露新增实现单元，保证 Android native package、CLI 和库级测试在 module-first 与 include-first 构建路径下拿到同一份 `mini` / follow 实现。

### 修复 (Fixed)
* 修复 Morse dash follow 语义：dash 的持续时间现在保存在 `sample_count`，`bit_count` 只推进一个 Morse 源元素，避免 Android lyrics/follow 在播放一个 dash 时把后续多个 dot/dash 提前标成 completed。
* 修复 Flash follow 连续性测试漏掉 `Collapse` 的问题，确保 `Litany` 和 `Collapse` 这类带 payload silence 的风格都会走允许 gap 的结构化 follow 回归，降低 Android visual / lyrics 高亮再次漂移的风险。
* 修复 `mini` decoder 内部命名债，将残留的 `MaxDecoder` 重命名为 `MiniDecoder`，避免库内实现名称继续泄露旧模式讨论过程。

### 弃用/删除 (Deprecated/Removed)
* 删除库层对旧 Flash 公开风格 `coded_burst`、`ritual_chant`、`deep_ritual` 的兼容口径，让 Android 设置、CLI 参数、C API 和 WAV metadata 都只面对 `steady`、`hostile`、`litany`、`collapse` 这一套新命名。
