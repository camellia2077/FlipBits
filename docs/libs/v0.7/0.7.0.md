## [v0.7.0] - 2026-05-14

### 新增功能 (Added)
* 确立 `0.7` 的版本目标：把 Flash 从“必须知道固定 style 才能 decode”的流程推进到更通用的逐 bit FSK 接收能力。目标是在 libs 侧建立 adaptive FSK receiver 的协议边界：当调用方没有提供 style，或者音频 metadata 缺失 / 不可信时，libs 可以先在候选 Flash style 之间遍历、估计 low/high 频率对、bit duration 和 gap pattern，再把可分辨的 low/high 映射回 `0/1` bitstream；当未来允许用户自定义 Hz、速度和间隙时，也能在 low/high 可区分的前提下提供解析基础。`0.7` 暂不把 live mode 作为交付重点，但要为后续手机麦克风实时接收、滚动窗口识别和 Flash live decode 留出 receiver 与诊断接口。
* 新增 Flash binary follow 的 `carrier_freq_hz` 字段，并贯通 `audio_core`、`bag_api`、C++ module 和 Android native include-first 边界，让每个 bit 的真实载波频率可以从 libs 暴露给 Android，而不再要求前端按 style 自己推导 Hz。
* 新增 Hostility、Collapse、Void 的 deterministic per-bit carrier schedule：Hostility 使用高区间上冲 jitter，Collapse 使用下坠 sag，Void 使用更低频的悲伤下坠 sag；这些 schedule 只依赖 bit position / byte index / bit index，避免 decoder 在拿到 payload 之前需要知道明文。
* 新增 Void 的 variable-window payload layout：Void 现在支持 `2.5x / 2.75x / 3x frame_samples` bit window、稀疏真实 payload silence 和 layout-aware decode，用于表达空虚、残缺和技术遗失后的低声悲伤，同时保留可解码边界。

### 技术改进/重构 (Changed/Refactor)
* 重构 Flash decode 分发，使 Hostility 和 Void 走 layout-aware decode、Collapse 继续跳过真实 silence、Zeal 继续使用专用 variable-window gap-aware decode；这为后续 adaptive FSK receiver 统一处理“变频、变长、gap”三类输入打基础。
* 调整 Flash signal info 生成逻辑，使 `bag_describe_flash_signal` 直接从 core payload layout 汇总 low/high carrier、bit duration、payload silence 和 decode path，供 Android 显示真实 Hz / timing / gap 信息。
* 调整 Hostility 与 Void 的 voicing 参数：Hostility 强化 edge、boundary accent 和上冲 carrier 以恢复尖锐愤怒听感；Void 降低 carrier 区间并弱化 accent、延长 release、增强低频空洞感，以区别于 Standard 的高效无感情通信。
* 扩展 API 与 sync 测试，覆盖 per-bit carrier follow、Void variable timing / sparse gap、Flash signal info 和跨 mode structured follow，确保 Android 侧消费的新 metadata 与 libs 实际渲染一致。
