## [v0.7.2] - 2026-05-16

### 技术改进/重构 (Changed/Refactor)
* 调整 `audio_io` 的 WAV metadata 与导出相关 ABI/实现细节，配合 Android 把生成音频的命名口径统一为 `mode[style]_content`，并把“创建时间”这类信息继续放回 metadata 侧处理，而不是再依赖文件名携带时间戳。
* 调整 `audio_core` / `audio_api` 的 mini 文本契约与错误映射，明确把 `space`、`\t`、`\r`、`\n` 视为可编码 separator，而把真正的空字符串单独归类为 `EMPTY_TEXT`。这次改动是为了解决 Android mini 模式下“纯空白或纯换行输入会闪一下后消失、难以判断是否真的生成”的排查问题，避免 UI、JNI 与 core 对“空输入”和“可编码空白输入”各自使用不同口径。

### 修复 (Fixed)
* 修复 Android 保存音频与导出音频在新命名规则下仍需要从文件名猜时间或样式的边界问题。现在 libs 会为 Android 侧继续保留更稳定的 metadata/状态信息来源，使保存库列表、导出结果和后续解析不必再把文件名时间戳当成唯一来源。
* 修复 mini 编码对纯空白输入的边界处理：`space`、`\t`、`\r`、`\n` 组成的文本现在会稳定生成 separator payload 与 silence PCM，不再因为 payload 被错误裁空而在 Android 侧落成 `ERROR_INTERNAL`。同时混合文本里的前导/尾随空白也会先按统一规则归一化，避免 Android 传入的 `mini` 输入在验证通过后又被 core 当成不可编码内容。
