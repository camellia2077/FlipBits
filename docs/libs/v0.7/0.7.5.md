## [v0.7.5] - 2026-05-25

### 新增功能 (Added)
* 新增 `bag.flash.phy_rules`、`bag.transport.encode_operation_steps`、`bag.transport.encode_work_plan` 和 `audio_io.wav_metadata_parse` 等 module-first 边界，把 Flash 规则、encode operation 阶段步骤、work-plan 计算和 WAV metadata 解析从原来的聚合实现中拆成可单独测试和复用的职责包。
* 新增 Android native package 可消费的对应声明/实现入口，让 Android 的非 module packaging lane 能继续通过 package-owned source owner 使用同一套 Flash 规则、transport operation 和 WAV metadata 解析语义。

### 技术改进/重构 (Changed/Refactor)
* 重构 `audio_core` 的 transport encode operation 实现，把 operation lifecycle、阶段步骤和 work-plan 计算分离到命名模块与 implementation unit；`transport_impl.inc` 不再同时承载生成入口、阶段推进、工作量估算和终态结果组装。
* 重构 Flash `phy_clean` 规则层，把配置归一化、layout/budget 计算和 voicing 选择规则收敛到 `phy_rules` 边界，保留 `phy_clean` 作为更薄的 encode/decode facade。
* 重构 Flash voicing 内部实现，把 support、texture 和 shell timing 拆成独立 include unit，并按依赖顺序组织，方便后续继续把 preset/timbre/cadence 规则迁移到更窄边界。
* 重构 `bag_api` result marshalling，把 lifecycle、raw format、follow marshalling 和 encode result copy 的包含顺序调整为显式依赖链，降低 Android package 直接 include `.inc` 时的顺序敏感性。
* 重构 `audio_io` WAV bytes metadata 解析规则，把 metadata parse helper 下沉到独立边界，同时保留 C ABI 和 Android fallback 能消费的非 module source/header。

### 修复 (Fixed)
* 修复 Android release native package 在 `RelWithDebInfo` 下直接编译 `audio_core` package-owned sources 时，Flash voicing shell、transport follow lyric lines 和 `bag_api` encode result copy 因内部声明顺序不可见而失败的问题。
* 修复 `follow_lyric_lines_impl.inc` 对 `TextTokenSpec` 的隐式包含顺序依赖，`follow_impl.inc` 现在先包含 tokenization，再包含 lyric line 和 payload timeline 逻辑。
