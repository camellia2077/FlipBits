## [v0.3.0] - 2026-04-19

### 新增功能 (Added)
* 新增基于正式 `audio_io_api.h` 的 Android WAV 编码/解码链路；App 现在会同时拿到 WAV 解码状态、metadata 解析状态与完整 WBAG metadata，可直接恢复 mode、flash voicing style、创建时间、时长、frame samples、PCM sample count、app/core version。
* 新增 Android 播放 runtime native bridge 与 Kotlin gateway，播放态现在具备统一的 `play / pause / resume / progress / scrub / complete / fail` 状态机能力，并能稳定计算 `elapsedMs`、`totalMs` 与 `progressFraction`。
* 新增 Android 子项目级 `Gradle` root 与本地 wrapper，`apps/audio_android/` 现在可以作为独立 Android Studio 工程直接打开、同步与构建。
* 新增 Android Kotlin 质量工具接线，包括 `ktlint`、`detekt` 与 AboutLibraries 资源导出流程，方便在 Android presentation 层单独做静态检查与发布前整理。

### 技术改进/重构 (Changed/Refactor)
* 重构 Android native audio I/O 边界，JNI 侧改为直接调用正式 `audio_io_api.h`，不再维护私有 `android_audio_io/audio_io_package.h` 协议；Rust CLI 与 Android 现在共享同一套 audio I/O contract。
* 重构 Android 侧解码 domain 模型，将旧的单一 `AudioIoCodes` 拆成 `AudioIoWavCodes` 与 `AudioIoMetadataCodes`，并在 `DecodedAudioData` 中显式区分“WAV 是否可读”和“metadata 是否可读”。
* 重构媒体库与导出链路，保存音频、导入音频、分享音频与 header 读取逻辑统一围绕正式 WAV + WBAG metadata 工作，不再依赖旧的私有解包包装层。
* 重构播放状态更新链路，将进度、scrub、resume-after-scrub、stop/completed/failed 等收敛到 `PlaybackRuntimeGateway` 与 `PlaybackUiState`，减少 UI 层手写边界判断。
* 重构 Android 构建配置，把 presentation 版本号从模块级 `build.gradle.kts` 抽到 `apps/audio_android/gradle.properties`，让版本号成为独立真源。

### 修复 (Fixed)
* 修复 Android 侧无法独立表达“WAV 可读但 metadata 缺失/损坏”与“WAV 本身损坏”两类错误的问题；现在导入、读取与解码流程可以分别处理两阶段状态。
* 修复播放 scrub 与越界进度在不同页面状态下容易漂移的问题；native runtime 现在统一做样本夹取、scrub 取消和完成后的收敛。
* 修复 Android Studio 需要依赖仓库根 `Gradle` 配置才能稳定同步和构建的问题；现在直接打开 `apps/audio_android/` 即可作为正式 Android 工程工作。
