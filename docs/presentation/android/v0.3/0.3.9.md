## [v0.3.9] - 2026-04-29

### 新增功能 (Added)
* 新增 `Pitch` 视图，并将可视化入口统一为 `Lanes / Energy / Pitch`。
* 新增 Lyrics 逐位点亮 (Bit-level Highlighting)，可直接标识当前音频对应 bit 位。

### 技术改进/重构 (Changed/Refactor)
* 移除 dual-tone 浅色/深色分支显色逻辑，统一颜色推导路径。
* 调整 dual-tone 颜色预览为 `background 50%` 与 `accent 50%` 组合，并保留描边表现。
* 重构 `AppThemeVisualTokens`，将 `outlineColor` 从装饰角色转为结构角色。
* 引入基于背景色推导的 `depthShadow`，统一替代原三色对比逻辑。
* 统一将 `subtleOutlineColor` 与 `visualizationInactiveToneColor` 映射到 `depthShadow`。
* 统一 Tab 未选中 icon 颜色为向背景色方向 `lerp` 混合，不再区分 `isDarkTheme`。
* 统一 Setting 与 Audio 未选中容器背景为 `backgroundColor`，不再混入强调色。
* 调整 Saved 列表文件夹名称颜色为 `onSurfaceVariant`，降低与主操作色竞争。
* 优化 Lyrics 布局密度：hex 每行最多 8 个、bin 每行最多 3 组（每组 8 bits）、文本最多 2 行。
* 优化当前播放 token 的居中显示，提升播放定位稳定性。
* 调整 token 容器显色规则，仅 Active token 保留 Accent 色调背景，Past/Future 统一 Transparent。
* 调整 Visual 波形渲染规则，窗口内已被播放头扫过的 bucket 在滑出前持续保持 accent 色。

### 修复 (Fixed)
* 修复未显式指定颜色文本未正确跟随 `MaterialTheme.colorScheme.background` 适配的问题。
* 修复 Saved 歌曲名在深色背景下默认黑色导致可读性不足的问题。
* 修复 Edit custom dual-tone 生成配色偏向单一亮度模式的问题，改为浅色/深色约 50% 概率。
* 优化 Android 播放进度轮询间隔从 `50ms` 到 `16ms`，缓解高亮与播放进度不同步感。
