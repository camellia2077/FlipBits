## [v0.4.0] - 2026-04-30

### 新增功能 (Added)
* 新增 `mini` Morse code 模式，并将传输模式整理为 `mini / flash / pro / ultra`：Android 现在可以直接编码 Morse-compatible 文本，支持小写自动规范化为大写、只接受 `A-Z` / `0-9` / 空格 / 常见标点，并在输入不符合 Morse 规则时显示更明确的错误提示。
* 新增 Morse speed preset：`Slow`、`Standard`、`Fast` 会影响 `mini` 编码速度，让用户可以在可听辨的慢速 Morse 和更紧凑的播放之间切换。
* 新增 Morse normalization 预览：输入区会展示 Morse 模式下的规范化文本，帮助用户在 encode 前确认大小写、空格和可接受字符如何被实际送入底层编码。
* 新增 Morse timeline visual：`mini` 模式不再复用 Flash/Pro/Ultra 的 hex/bin 视觉，而是用朴素 block timeline 展示 dot/dash 与间隔，方便用户直接看见 Morse 节奏结构。
* 新增 Morse lyrics / follow 适配：播放跟随可在 Text lyrics 与 Morse lyrics 间工作，Text 视图按规范化字符/词跟随，Morse 视图按 dot/dash token 跟随。
* 新增 Flash 四种情绪风格：`Steady`、`Hostile`、`Litany`、`Collapse` 取代旧 `coded_burst` / `ritual_chant` / `deep_ritual` 文案，分别面向日常平稳、敌意攻击、肃穆吟诵和恐惧结巴的声音表达。
* 新增 Flash 情绪音效表现：`Litany` 具备更低沉的 `220 / 440 Hz` 频率、周期性吟诵停顿、空格/标点大停顿、低频 drone、mechanical throat、chant swell 以及三次开场钟击和一长一短结尾钟击；`Hostile` 使用 `450 / 900 Hz` 的尖锐攻击感；`Collapse` 使用 `280 / 560 Hz` 并加入不规则 hesitation / stutter silence。
* 新增 Flash visual 对静音停顿的表达：Flash 波形图现在会把真实 silence 显示为空白，不再在停顿区继续画 pitch 线，`Litany` / `Collapse` 的停顿能直接在视觉上区分出来。
* 新增 Hex follow 的 4-bit 展示：播放到当前 hex token 时只显示当前 token 对应的 4-bit bin，并按当前 bit 高亮，让用户更直观看到 hex 与 binary 的转换关系。
* 新增 Max 改名后的产品口径：原本讨论中的 `max` Morse code 被收束为 `mini`，Android UI、样例、保存筛选、decode/encode request 和 native bridge 均使用新的模式命名。

### 技术改进/重构 (Changed/Refactor)
* 重构 Android mode wiring，使 `mini / flash / pro / ultra` 在 UI option、native gateway、saved audio metadata、sample catalog、测试和文案中共享同一套模式口径。
* 重构 Flash style preset，保留一个用户可见选择器，但每个 preset 内部明确携带 `signalProfileValue` 与 `voicingFlavorValue` 两轴，为后续继续打磨不同情绪声效保留边界。
* 重构 Flash visual 的分析逻辑，不再按已删除的旧 profile magic value 判断 active window，而是通过 `FlashVoicingStyleOption` 的语义 helper 推导视觉窗口。
* 调整 Playback follow annotation mode：`mini` 只暴露 Morse follow 选项，不再显示 Hex / Bin；`flash / pro / ultra` 继续保留 Hex / Bin。
* 调整 Lyrics 高亮行为，当前 token 闪烁时已完成 token 保持静态 completed，避免 `Litany` 这类长停顿风格导致已经扫过的 hex/bin 一起闪烁。
* 调整 Flash pitch visual，移除不必要的背景竖线，增大 low/high 的高度差，并在 silent gap 中断开线条，让用户更容易识别当前风格的节奏和停顿。
* 调整 Flash 文案与多语言资源，所有 `values-* / strings_audio.xml` 同步新增 `mini`、Morse、Flash 四情绪与速度相关文案，避免翻译 key alignment 失败。
* 放宽 Android detekt 配置中过于严格的风格规则，并补充 `ktlint-check`、`detekt`、`assemble-debug` 等验证覆盖，使本轮较大 UI/native 接线改动能稳定通过质量入口。

### 修复 (Fixed)
* 修复 Morse dash follow 会推进过多高亮位的问题：dash 现在只推进一个 Morse dot/dash 元素，避免 Android 播放到 dash 时把后续多个 Morse 元素提前标记 completed。
* 修复 Flash follow 结构化测试漏掉 `Collapse` 的问题，确保 `Collapse` 的 irregular hesitation silence 也受 payload gap 连续性测试覆盖。
* 修复 Morse UI 文案硬编码问题，将 normalization 预览、错误提示和 speed preset 文案纳入 Android string resource 体系，避免多语言环境漏翻译。
* 修复 Flash visual 中残留已删除 profile 值的问题，避免维护者误以为 Android 仍存在旧的第三种节奏 profile。
* 修复 `Collapse` 旧结尾音残留和过强背景噪声问题，移除会压过主体 tone 的 instability layer，并改为更贴近恐惧/结巴主题的 silence hesitation。

### 弃用/删除 (Deprecated/Removed)
* 删除 Android 用户可见的旧 Flash 风格 `coded_burst`、`ritual_chant`、`deep_ritual`；旧保存值会迁移到新的四情绪口径，后续 UI 只展示 `Steady`、`Hostile`、`Litany`、`Collapse`。
* 删除 `mini` 模式下的 Hex / Bin annotation 入口；Morse code 只显示 Morse 专用 follow，避免把点划节奏误包装成普通 payload 进制视图。
