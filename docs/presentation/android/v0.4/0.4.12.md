## [v0.4.12] - 2026-05-14

### 新增功能 (Added)
* 新增 Flash `Hz` visual 模式，用来消费 libs 暴露的 per-bit `carrier_freq_hz` follow 数据，直接展示每个 bit 的真实载波频率；这个 Android 改动依赖 libs `0.7` 新增的 binary follow carrier 字段，避免前端继续按 style 猜测 low/high Hz。
* 新增 `Hz` visual 的动态频率窗口和当前 Hz HUD：各 Flash style 会使用自己的 Hz range，播放到 gap 时 HUD 显示 `-- Hz`，播放到 voiced bit 时显示当前 bit 的真实 Hz，让 Zeal、Hostility、Collapse、Void 的频率变化可以被直接观察。
* 新增 Flash visual 顺序 `Lanes / Pitch / Pulse / Hz`，并删除旧 `Tone` 入口；`Hz` 作为英文保留字不翻译，和 `flash`、`emoji` 这类协议/产品 token 保持同一口径。

### 技术改进/重构 (Changed/Refactor)
* 重构 Android Flash follow 数据链路，贯通 JNI、DTO、保存音频 decode cache 和 visual model，使 saved audio 与实时生成结果都能保留并恢复 libs 提供的 per-bit carrier frequency。
* 调整 Flash `Lanes` visual：保留 bit 间真实 gap，并把整块离散跳色改为播放竖线扫描到的位置才变色，使 visual 更接近“当前播放点正在读取 bit”的语义。
* 调整 `Hz` visual 的 style 标尺以匹配 libs `0.7` 的新 carrier schedule：Hostility 使用上冲高区间，Collapse 使用下坠 sag，Void 使用更低的 `180-230 / 360-460 Hz` 区间；Android 侧只消费 libs 暴露的真实 per-bit Hz，不再自维护一套独立推导。
* 扩展 Flash adb/debug scenario、visual perf trace 和播放窗口状态，方便用实机自动化捕获 visual 对齐、暂停漂移、gap 显示和不同 playback speed 下的回归数据。

### 修复 (Fixed)
* 修复 Flash visual 在暂停后仍继续向左平移的问题，统一播放位置提交与 visual window 更新，确保 `Lanes`、`Pitch`、`Pulse`、`Hz` 停在用户点击暂停时的实际播放点。
* 修复 `Lanes` / `Pitch` 播放过程中出现不必要跳动的问题，将 visual window 与 playback sample position 对齐，并通过 debug trace / adb capture 保留可复查证据。
* 修复 Litany / Collapse 等带长停顿 style 在 Hz visual 中显示不连续、跳动或误把 gap 当成 tone 的问题，改为优先使用 libs follow timeline 的 voiced bit 和 silence gap 边界。
