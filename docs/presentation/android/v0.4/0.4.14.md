## [v0.4.14] - 2026-05-16

### 新增功能 (Added)
* 新增 Android `Material` 主题自定义配色能力，并把 dual-tone 主题复用到复制按钮/文案颜色与固定图标颜色配置上。这样 `Settings` 现在既能细调 Material 颜色，也能让 dual-tone 主题在复制、说明和主题编辑入口里保持统一视觉。
* 新增 mini Visual 页面专用的 `MiniVisualPerf` 调试观测能力，并补齐 `DEBUG_MINI_SCENARIO` 对 `wb.display=visual`、`wb.visual.perf_overlay=true`、`MiniAlignmentPerf` / `MiniVisualPerf` 的自动化抓取链路。这样后续 agent 可以直接用 adb 场景复现实机播放并对比 mini Morse visual 的平滑、窗口跳变和绘制指标。

### 技术改进/重构 (Changed/Refactor)
* 调整 `Settings` 的主题配置交互，把 theme 折叠展开状态做成持久化，包括 dual-tone theme 的各组展开状态。这样用户离开页面或重启应用后，主题编辑器会回到上次的展开/折叠位置，而不再每次重新展开全部设置。
* 调整 mini 速度选择 UI：从互斥 segmented button 改为和 flash 一致的下拉选择器 `Speed style`，默认 `standard`，并把选择结果持久化为“只有首次进入才回落到 `standard`”。这样 mini 与 flash 的 style 选择入口更一致，切换模式后也不会频繁丢失速度偏好。
* 调整 Android 生成音频的保存/下载命名规则为 `mode[style]_content`；仅 `flash` 与 `mini` 会带 `[style]`，不再把时间戳写进文件名，列表中的生成时间改为优先从歌曲 metadata 解析。这样保存音频、导出音频和后续库解析统一走一套命名口径，不再把文件名时间当成核心数据源。
* 重构 mini 的 Morse token 与 visual 表达：Morse 行改为固定 dot/dash 图形单元，不再用纯文本 `.-`；token 行也从固定 `chunked(6)` 改成“单行最多 6 个 + 按宽度装箱”。这样字母、数字和更长的 Morse 片段都能按实际宽度稳定换行，视觉节奏也更接近真实 Morse 单元。
* 调整 dual-tone 主题下的跟随线条颜色语义：token 竖线 divider 统一走 outline color，Visual `Lanes` 的横线也改用 outline。这样 dual-tone 模式下 token 边界、可视化导线和整体轮廓语义更一致，不再混用与主题不匹配的默认分割色。
* 调整 `Settings` 的 sample 配置入口，新增可折叠的 `Sample text` section，并把 `Auto-fill sample text` 与 `Sample decoration` 归并到同一组里，同时把 section 展开状态和自动填充偏好一起持久化。这样示例文本相关设置不再散落在配置页里，用户关闭自动填充后，重启应用或重新进入页面也会保持上次选择。

### 修复 (Fixed)
* 修复 Audio 页“无输入也能点生成/空白输入被错误拦截”的行为边界。现在真正的空字符串会提示用户“请输入文本，或使用随机文本”，并阻止生成；仅包含空格、换行等可编码空白的输入则继续允许生成，`mini / flash / pro / ultra` 行为保持一致。
* 修复 flash 歌词页点击 token 后不够稳定地展开并定位当前 token 的问题。现在点选 token 展开后会直接对齐到当前播放 token，减少长句或多 token 场景下“展开了但没对上当前位置”的感受。
* 修复 mini Morse 可视化在数字/长码内容下的排版和观感问题。Morse token block 现在不会再因为内容变长而退化成一列一个，也不会继续依赖固定字符数粗暴换行。
* 修复 mini Visual 调试链路缺少可量化播放信息的问题。现在开启 visual perf overlay 后，不会盖住 Morse timeline 本体，同时能持续输出 `drawAvgMs`、`rawUpdate/s`、`rawStepMaxMs`、`smoothStepMaxMs`、`visualErrorMs`、`windowStepMaxMs` 这类核心指标，便于实机判断“卡顿”到底来自绘制、原始播放位置更新还是窗口平移。
* 修复 Android mini 模式对纯空白输入的生成体验问题。之前用户输入只有空格、换行，或“前导空格 + ASCII + 回车”这类文本时，点击 `Create` 后 mini player 可能只短暂闪现又消失，既无法稳定生成音频，也难以从界面判断到底是验证失败、编码失败还是结果被清空。现在 Android 端会把 mini 示例文本和真实输入统一走同一套归一化/验证链路，并配合 libs 新契约把可编码空白稳定落成音频；真正的空字符串才会明确提示用户需要输入文本或使用随机示例。
