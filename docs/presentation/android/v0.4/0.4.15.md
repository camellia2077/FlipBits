## [v0.4.15] - 2026-05-17

### 技术改进/重构 (Changed/Refactor)
* 统一 flash 风格中 `Hostility` 的 Android 公开命名，把 `FlashVoicingStyleOption`、wire 常量、JNI 枚举映射、设置页文案和对应测试全部改成 `Hostility`。这样 `Settings`、JNI/native 桥接和后续 metadata 口径不再混用 `Hostile` / `Hostility` 两套名字。
* 重构多处 Compose UI 装配层，但保持现有交互和 public API 不变：`ConfigThemeAppearanceSection` 收拢了 dialog state、brand theme expand policy 和派生 render state；`PlaybackFollowAnnotationRows` 收拢了 follow annotation 的 byte/morse render state；`AudioInputActionsCard` 收拢了 selector/encode action render state；此前已整理的 `AudioFlashSignalVisualizer`、`FlashSignalVisualizationState`、`PlaybackDisplaySection` 也继续沿“主 Composable 变浅、state/effect/render decision 更集中”的方向保持同文件浅分层。这样后续调整主题配置、follow annotation、输入卡片和 visualizer 装配时，不必再在同一个入口函数里同时追踪状态、分支和渲染细节。
* 调整 Kotlin LOC 重构辅助工具，扫描前会先清掉对应语言的旧 detail logs，并把 Compose UI 重构 prompt 更新为更贴合当前 Android 实践的版本，明确要求默认保持行为等价、保持 public API 稳定，并优先抽 `derived state / render policy / dialog state / expansion policy`。这样重新跑 `python tools/scripts/loc/run.py --lang kt --responsibility-risk` 时，不会再混入过期的 `P2` 明细结果，后续 Android Compose 重构也能直接复用更贴近项目现状的提示词。
