## [v0.4.16] - 2026-05-19

### 新增功能 (Added)
* 新增 `Material` 自定义配色的完整编辑能力：`Edit custom colors` 入口统一改为 `Edit colors`，编辑页新增 `Preset name` 命名字段，并支持删除当前颜色 preset。这样用户现在可以直接在设置里重命名、维护和清理自己的 Material 自定义颜色集合。
* 新增 dual-tone 自定义颜色的长按排序能力，可直接通过长按调整 custom color 的显示顺序。这样常用颜色可以手动排到更靠前的位置，减少反复查找。
* 新增 Morse visual 的 `Horizontal` 横向布局，并提供与 `Vertical` 互斥的胶囊切换按钮。这样 mini Visual 除了原来的纵向展示，还能按不同观看习惯切到横向模式。
* 新增 4 个 dual-tone 主题：`Brass Forge`、`Fires of Fate`、`Ecstatic Rapture`、`Plague Mire`，并同步更新对应的主题命名与描述口径。这样 dual-tone 主题库补进了新的高辨识度配色，旧主题中与当前颜色不匹配的描述也一并清理。

### 技术改进/重构 (Changed/Refactor)
* 调整 `Settings` 的主题配置持久化策略，把 `Theme appearance` 的 `Material` / `Dual-tone` 选择、Material 当前选中色、light/dark 各自上一次使用的颜色，以及 Material custom palette、built-in palettes 和各个 built-in family 的折叠状态全部做成持久化。这样用户切换主题模式、重启应用或重新进入设置后，都会回到更接近上次操作的位置和配色。
* 调整 Material 自定义颜色的前景色和随机色逻辑：custom edit 里的前景色现在统一按当前 `primary` 明暗实时选择高对比前景，随机色也会按当前有效 light/dark 模式分别避开过浅或过深区间，同时把齿轮预览一并接入随机后的实时颜色。这样 Material 自定义主题在编辑、随机和预览时的可读性与视觉一致性更稳定。
* 调整 Material / dual-tone 的主题切换体验，为 dual-tone 主题切换、Material 颜色切换，以及 `Material <-> Dual-tone` 模式切换补上过渡动画。这样不同主题体系之间切换时不再那么生硬。
* 调整 Material palette 与 dual-tone custom 的布局结构：dual-tone custom section 拆成双层 header，built-in palette 改为双列卡片，family header 左侧新增代表色圆点，palette 卡片不再重复显示 family 名字，颜色名支持两行省略，`Selected` 徽标改成更适合半宽卡片的纵向布局。这样多语言和窄宽度场景下的设置页排版更稳定，信息密度也更合理。
* 调整 Material custom 导入流程，使其与 dual-tone 一样处理重复项：单条导入命中已有同配置 preset 时，不再直接跳过，而是先弹出 `Overwrite` / `Add copy` 确认。这样重复导入时用户可以明确选择覆盖现有 preset，还是保留原项并另存一份副本。
* 调整 mini Visual 中未播放 Morse 段的视觉 token，提炼 `MorseInactiveToneAlpha` 并让未播放 Morse 符号统一改用 `visualizationInactiveToneColor.copy(alpha = MorseInactiveToneAlpha)`。这样未播放部分与已播放部分的层次关系更清晰，整体观感更顺。
* 调整齿轮图标在 Material 主题下的配色语义，让 `primarySplit`、`secondarySplit` 和 `outline` 都随当前 Material 颜色与明暗模式变化。这样设置页里的齿轮预览终于会和当前 Material 主题一起变化，而不是停留在固定配色。
* 调整 mini player 里 `Visual > Hz` 的视觉口径：`Hz` 参考线现在与 `Lanes / Pitch / Pulse` 统一使用同一套 outline 透明度，右上角当前频率 HUD 也改成按最大 `Hz` 数字预留固定宽度，并把 `Hz` 单位固定在右侧。这样 `Hz` 模式不会再比其它 visual 更深，当前频率在数字位数变化或 `--` 切换时也不会再频繁伸缩闪动。
* 调整 `Visual > Hz` 的 `low/high` 两行标签配色，改为按 mini player 背景取对比前景色，而不是直接跟随 `primary/secondary` 槽位。这样不同 dual-tone / Material 主题下的上下参考标签可读性更稳定，不会再出现和背景过近的情况。

### 修复 (Fixed)
* 修复 Material custom edit 在随机颜色后仍沿用旧 `primary` 前景色的问题。现在随机出新主色后，前景对比色会立刻按新的 `primary` 明暗重新计算。
* 修复 custom 颜色导入时缺少具体校验反馈的问题。现在导入失败会给出更明确的检验内容提示，用户能更快判断是格式不对、字段缺失还是内容不合法。
* 修复切换 light/dark 模式后颜色选择容易丢失上下文的问题。现在两个模式都会记住各自上一次使用的颜色，不再在明暗模式切换后回退到不相关的颜色。
* 修复切换语言时先短暂黑屏、再完成语言切换的问题。现在语言切换过程的界面过渡更平滑，不再先闪一段黑屏。
* 修复应用冷启动时会先闪一下默认 dual-tone 主题、再切回上次主题的问题。现在初始 `uiState` 与 DataStore 中的主题恢复路径保持一致，冷启动不会再先露出默认主题。
* 修复 `Visual > Hz` 在连续播放/暂停切换下的画面跳动问题，收紧了 debug playback end-action 与 visual trace 的状态重置链路。这样高频 `play/pause` 调试场景里不再因为错误重启播放或沿用旧 trace 基线而放大出一次性大跳变。
