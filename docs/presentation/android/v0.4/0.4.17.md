## [v0.4.17] - 2026-05-20

### 新增功能 (Added)
* 新增 `untranslated-equals-english` 检查流程，专门用于 `de`、`es`、`fr`、`it`、`pl`、`pt-rBR` 这类拉丁字母语言，直接找出仍与英文完全相同的文案，并按 `keep_en`、`needs_translation`、`needs_context` 输出 JSON-first 任务结果。这样 agent 不必再硬扫整份 XML 才能定位整句漏翻。
* 新增 `mixed-language-context-audit`，把 `zh`、`zh-rTW`、`ja`、`ko`、`ru`、`uk` 等语言里残留的英文按上下文分桶，并额外输出缺少 `CONTEXT` 注释的条目。这样“查哪里还没翻”和“查哪些地方缺上下文”现在是两条独立但衔接的流程。

### 技术改进/重构 (Changed/Refactor)
* 重构 Android 多语言翻译工具的统一策略源，把锁定英文、保留原生语言名、locale 同形合法词和 prompt 共享术语全部收口到同一份 policy，并让 `lint`、`mixed-language`、`audit` 与 prompt 生成共享这套定义。这样术语保留规则不再分散在代码、文档和 prompt 里各写一份。
* 重构翻译入口边界：项目入口统一收敛到 `apps/audio_android/AGENTS.md`，执行步骤统一放在 `.agent/workflows/translations`，工具定义统一放在 `tools/repo_tooling/android_translate/docs`。这样 agent 看项目、走流程、查工具定义时不再在多个入口之间来回跳转。
* 调整翻译任务输出，强化 `JSON first, Markdown optional`。`compare`、`key-alignment`、`mixed-language`、`mixed-language-context-audit` 等任务 JSON 现在统一带 `execution_contract`，并补充 `decision_hint`、`ui_surface`、`length_pressure` 等执行字段。这样 agent 可以更直接地按结构化任务改 XML，而不是重新解析整份报告。

### 修复 (Fixed)
* 修复 Android 多语言资源里多批整句英文漏翻和锁定术语缺失问题，补齐 `ja`、`ko`、`zh`、`zh-rTW`、`ru`、`uk`、`de`、`es`、`fr`、`it`、`la`、`pl`、`pt-rBR` 的残留英文、dual-tone 主题描述和 `Tokens` 等保留词口径。这样主要 locale 的“英文未翻”与 mixed-language 审计结果已收敛到干净状态。
* 修复英文基线里一批缺失的 `CONTEXT` 注释，并同步清理 workflow 与工具文档中的旧目录、旧入口和旧流程表述。这样后续翻译任务在判断 UI 位置、长度压力和是否允许保留英文时，有更稳定的上下文可用。
