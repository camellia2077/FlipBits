## [v0.4.6] - 2026-05-08

### 新增功能 (Added)
* 新增 Android 翻译 workflow 入口，将 app 文案翻译与示例文本翻译拆分到 `.agent/workflows/translations/`，并让 `apps/audio_android/AGENTS.md` 只保留薄索引。
* 新增翻译工具的 locale prompt profile 文件，覆盖 `de`、`es`、`fr`、`it`、`ko`、`la`、`pl`、`pt-rBR`、`ru`、`uk`、`zh`、`zh-rTW` 等语言的本地化约束。
* 新增 translation key alignment 的结构化 `*.task.json` 输出，让缺 key 修复可以直接读取机器可解析任务，而不是依赖手工解析 Markdown 报告。

### 技术改进/重构 (Changed/Refactor)
* 调整 Flash Visual 可读性，将 viewport 收窄到 `0.80s`，并在左右各约两个 bit 范围内做边缘渐淡，减少同屏 bit 密度但不改变播放时间轴和进度语义。
* 优化 Flash Visual 暂停行为，让 smoother 在暂停时保留当前视觉 sample position，避免暂停瞬间因协程重启或 raw anchor 对齐造成左右瞬移。
* 优化 Lyrics 长音频跟随性能，将 token/line timeline 查找从线性扫描改为二分查找，并在大跨度 seek 时直接落到目标行，避免 LazyColumn 跨大量歌词行播放动画导致卡顿。
* 精简输入区编码规则布局，将 `chars`、`bytes` 与输入编码摘要收敛到一行，并把可点击的 Input rules 移到信息图标按钮，避免同一区域重复显示规则文本。
* 重构 Android 翻译工具 prompt profile 加载方式，从硬编码 Python profile 改为读取 `prompts/locales/*.md`，方便后续为不同语言补充定制化翻译要求。
* 更新项目 README 定位，明确 Morse code (`mini`)、逐 bit BFSK / FSK (`flash`)、DTMF-like 双音映射 (`pro`) 和 `16-FSK` (`ultra`) 的技术命名、效率差异、可视化学习价值与 `flash` 的二进制 low/high 听感来源。

### 修复 (Fixed)
* 修复长音频 Lyrics 模式拖动进度条或跳转时 token 块移动明显卡顿的问题。
* 修复 Visual 播放中暂停时波形图可能向左或向右瞬移的问题。
* 修复输入区 `flash uses utf-8 text` / `Input rules` 等规则说明重复出现的问题，并同步更新多语言 XML key。
