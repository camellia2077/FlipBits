## [v0.4.7] - 2026-05-09

### 新增功能 (Added)
* 新增 Android 翻译 lint baseline 机制，支持通过 `python tools/scripts/android/translate/run.py lint --write-baseline --baseline-file <path>` 生成历史问题基线文件（新增 `tools/scripts/android/translate/lint-baseline.json`）。
* 新增翻译 lint 的“仅拦新增问题”模式，支持 `--fail-on-new` 配合 `--baseline-file` 在保留历史遗留问题的前提下阻断新引入问题。

### 技术改进/重构 (Changed/Refactor)
* 重构翻译工具 `lint` 命令参数与执行路径，在 `tools/scripts/android/translate/run.py` 增加 baseline 读写与问题过滤分支，并在 JSON 输出中补充 `fail_on_new`、`new_issues`、`baseline_file` 等诊断字段。
* 重构翻译 lint 内核（`tools/scripts/android/translate/commands/translation_lint_and_autofix.py`），新增 issue fingerprint 生成、baseline 读写、基线差异过滤能力，为后续 CI 渐进治理历史本地化问题提供稳定接口。
* 调整 Android 构建门禁策略（`apps/audio_android/app/build.gradle.kts`）：
* `translationLintWarn` 挂载到 `preBuild/check`，本地与 debug 流程默认告警不阻塞。
* `translationLintStrict` 切换为 `lint --fail-on-new --baseline-file ...`，并挂载到 `assembleRelease`，release 只对新增翻译问题阻塞。

### 修复 (Fixed)
* 修复 `assembleRelease` 挂钩在某些任务图下 `tasks.named("assembleRelease")` 可能报 task not found 的兼容性问题，改为 `tasks.matching { it.name == "assembleRelease" }.configureEach` 惰性绑定。
* 修复翻译 lint 全量阻塞导致 release 长期不可用的问题，改为 baseline 首版落地后仅阻断新增问题，历史问题可分批清理。
