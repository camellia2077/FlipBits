## [v0.4.8] - 2026-05-10

### 新增功能 (Added)
* 新增 Flash debug scenario 入口，支持通过 adb 指定 Flash style、Visual 模式、输入文本、UI/headless 场景和短时播放时长，方便真机回归测试与 agent 驱动诊断。
* 新增 Mini debug scenario 入口，支持通过 adb 覆盖 slow、standard、fast 三种 Morse speed 的 Visual/Lyrics 对齐采集预工作。
* 新增 Flash/Mini 真机自动化文档与日志过滤脚本，覆盖 `FlashAutomation`、`FlashAlignmentPerf`、`FlashVisualPerf`、`FlashLyricsPerf`、`MiniAlignmentPerf` 等 adb 诊断标签。
* 新增 debug-only Visual/Lyrics 统一对齐日志，让 Flash 与 Mini 可以在同一条采样记录中对比 audio sample、visual readout、8-bit 行和 Lyrics token 状态。
* 新增 Custom dual-tone 纯文本导入导出，支持单个主题导入、批量主题导入、重复主题确认、单个主题导出和全部 Custom 主题导出。
* 新增首次进入 Audio Flash 输入框的 sample emoji，并让 Sample emoji 开关只移除或恢复 app 自动添加的 emoji 前缀，不改动用户手写 emoji。
* 新增 Android README 的本地化语言说明，明确当前支持的界面语言并将 Latin 标注为 dog Latin。

### 技术改进/重构 (Changed/Refactor)
* 重构 Flash Visual 时间源，让 Canvas、8-bit 行和 Lyrics 对齐诊断共用 Flash Visual 内部平滑 sample 状态，避免多套时间源造成主观延迟。
* 调整 Flash Lanes/Energy 绘制策略，保持竖条间距与竖条宽度稳定，并按 bit 持续时间增加同一 bit 内竖条数量；Pitch 继续用横条长度表达持续时间。
* 调整 Mini Visual 表现，与 Flash 一致使用固定 playhead，并移除边缘竖条渐淡效果，让播放前后状态的颜色语义更统一。
* 优化 Flash Litany 的边缘淡化策略，避免慢速 style 因同屏 bit 数少而让最左侧竖条显得过淡。
* 重构 Custom dual-tone 编辑体验，将导入、分享和 Add 放到 Custom 分组头部；编辑弹窗内保留随机、复制配置和删除操作，并让输入框颜色过渡更平滑。
* 调整 Custom dual-tone 配置格式，移除版本头，使用更容易手写的 `name`、`primary`、`secondary`、`outline` 键；导入时忽略空行和首尾空格，key 不区分大小写，`#` 可选。
* 统一 Android 命名与图标语义，将遗留 `WaveBits` 产品名迁移为 `FlipBits`，并区分本地保存、保存到 app、导入音频、导入主题和分享主题等图标含义。
* 调整 Android debug APK 安装流程，自动先卸载测试设备上的旧包再安装标准 `app-debug.apk`；`FlipBits-*` APK 命名保留给 staging/release 产物。
* 更新翻译工具与 agent 文档，收敛 translation workflow 入口、locale prompt 说明、lint/autofix 行为和绝对路径中的 FlipBits 命名。
* 更新 Android quality 本地入口与 Kotlin policy，让 `python tools/run.py android quality` 覆盖 ktlint/detekt，并修复 release Kotlin 编译中的 nullable scenario 访问。

### 修复 (Fixed)
* 修复 Flash Void/Litany 在 Lanes/Energy 竖条向左滑出时宽度被压缩、没有直接离屏消失的问题。
* 修复 Flash Visual、8-bit 行和 audio 主观不同步的问题，将 8-bit 行从独立读取状态改为跟随统一 visual sample。
* 修复 Flash Lyrics token tape 难以通过 adb 判断对齐的问题，补充 `FlashLyricsPerf` 与 `FlashAlignmentPerf` 诊断输出。
* 修复 Mini Visual 与 Lyrics 缺少统一采样日志的问题，补充 `MiniAlignmentPerf` 用于 slow、standard、fast 真机数据采集。
* 修复 Custom dual-tone config 复制使用旧 Compose Clipboard API 的 deprecation warning。
* 修复 Custom dual-tone 导入导出新增文案后的多语言 key alignment 与 translation lint 问题。
* 修复 `:app:compileReleaseKotlin` 因 debug scenario nullable smart-cast 不成立导致的 release 编译失败。

### 弃用/删除 (Deprecated/Removed)
* 删除临时 `temp/format_flash_visual_perf.py`，改用正式的 Flash/Mini adb 日志过滤脚本。
* 删除遗留 `cmake/wavebits_core_version.cmake` 与 `cmake/wavebits_test_helpers.cmake`，避免继续暴露旧项目命名。
