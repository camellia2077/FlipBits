## [v0.4.9] - 2026-05-11

### 新增功能 (Added)
* 新增 Debug 场景对 `flash` 自动化输入源的精确控制，支持通过 `sampleLength` / `sampleId` 直接选择内置 short/long 样本文本，并在 logcat 中输出输入来源、字符数与 payload bytes，方便复现长文本音频生成问题。
* 新增 `mini` / `pro` / `ultra` 编码进度采集场景，支持 adb 传入模式、样本长度、重复次数、Mini speed、采集时长和轮询间隔，并持续记录进度条可见性、phase、percent、busy、cache 和生成结果落态。
* 新增生成音频的 Decode/Roundtrip 回归覆盖，验证所有传输模式都能从刚生成的 PCM 解码回 payload，并保留 follow data 与状态文案。

### 技术改进/重构 (Changed/Refactor)
* 调整 Flash 长 cadence 的分段策略，`Litany` 等长间隔 voicing style 使用更小的 payload segment 上限，降低单段生成时长过长导致失败落态或 0s 显示的风险。
* 改进分段编码进度聚合，跨 segment 时不再反复回跳到 `PreparingInput`，而是保持 `PreparingInput -> RenderingPcm -> Finalizing` 的整体进度语义。
* 增强 Debug 日志覆盖，在生成缓存、播放命令、Flash UI 播放步骤和 follow/cache/result 状态中记录 source、sample count、file-backed 状态和 mini player duration，便于区分 Generated 与 Saved 播放源。
* 将播放器详情页默认展示从 Visual 调整为 Lyrics，并仅保留 Lyrics / Visual 切换项，让文本跟随和 payload 不可用状态更直接暴露。
* 更新 Android 自动化文档入口，在 `AGENTS.md` 与 `README.md` 中补充自动化覆盖总览和 encode progress 采集文档。

### 修复 (Fixed)
* 修复编码失败后仍暴露旧生成音频或 0s mini player 的落态问题，失败时会清空 stale generated PCM、metadata、follow data、flash signal info 与 generated playback 状态。
* 修复 Saved 音频播放源在后续 Generated 编码失败后被错误覆盖的问题，保留当前 Saved mini player 与播放源，避免播放/暂停命令落到错误音频。
