## [v0.5.0] - 2026-05-21

### 新增功能 (Added)
* 新增输入样例长度选择的持久化恢复：首次安装或没有历史值时仍默认 `Short`，用户切到 `Long` 后会写入 DataStore，后续重新进入应用会恢复上次选择，不再每次强制回到 `Short`。
* 新增 Settings 自定义 Material palette 与 dual-tone theme 的横向滑动删除交互。短滑会停在 reveal 状态并露出右侧全高删除槽，长滑超过阈值后松手会直接删除，删除区也从窄胶囊改为整行等高的右侧 action lane。

### 技术改进/重构 (Changed/Refactor)
* 重构 mini player 打开的播放器 detail 页面，把展开态播放器改成更接近主流音乐播放器的全屏 player surface：顶部固定为左侧折叠、居中模式文本、右侧分享/下载动作，播放显示区与底部时间轴/传输控制分层布局，播放列表弹窗作为独立覆盖层接入。
* 调整播放器 detail 的垂直空间分配，减少底部 dock padding 和 transport secondary actions 间距，并把互斥控制移动到更靠上的位置，为 visual、lyrics 和 token follow 区域留出更多可用高度。
* 调整 mini player 底部播放列表弹窗的信息密度，只显示文件名本体，不再显示扩展名、模式、时长、来源、采样率和时间等附加信息，并让弹窗容器跟随当前主题配色。
* 调整 Settings 里 dual-tone 内置主题的复制按钮文案，从 `Copy config` 收敛为 `Copy` 并同步多语言资源，避免部分语言翻译后文本过长，把主题颜色名挤压成竖排。
* 调整 Audio 页样例长度切换逻辑：只有开启自动填充样例文本时，切换 `Short` / `Long` 才会刷新输入内容；关闭自动填充时只保存长度选择，不再改动用户当前输入。

### 修复 (Fixed)
* 修复点击上一首/下一首切换已保存音频时，上一首音频仍继续播放的问题。现在切换播放源会正确停止旧播放会话，避免两个音频状态重叠。
* 修复 Flash Visual/Lanes 首次播放时 viewport 先左移、再短暂右跳、再恢复左移的问题。Flash visual 的平滑播放位置状态现在会在启动阶段持续存活，只更新 raw anchor 数据，不再因为粗粒度播放进度回报而重建 smoother。
* 修复 Ultra Binary/Hex token 页播放启动时当前 bit 在第 1 位和第 5 位之间快速跳动的问题。播放进度插值层现在会在非 seek 的正常播放中忽略 `AudioTrack` 短暂回退读数，同时仍允许真实 seek 通过 `seekVersion` 回到更早位置。
* 修复 Flash visual 快速播放/暂停后暂停画面继续向前瞬移的问题。暂停态不再继续用 raw playback sample 校正 visual sample，而是保持最后一帧 visual sample；只有用户拖动进度条时才允许 snap 到 raw，实现“播放竖线在哪里，暂停后就停在哪里”。
