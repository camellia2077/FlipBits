## [v0.2.6] - 2026-05-15

### 技术改进/重构 (Changed/Refactor)
* 收口 Rust CLI 的 FFI `unsafe` 边界：将 `apps/audio_cli/rust/src/audio_io_api.rs`、`bag_api.rs`、`util.rs` 里的裸指针读取、native buffer 释放、decoder/job 生命周期管理集中到局部 guard 和 helper，避免 `unsafe` 继续散落在 encode/decode 主流程里。
* 为 Rust CLI crate 增加 `#![deny(unsafe_op_in_unsafe_fn)]` 和 `#![warn(clippy::undocumented_unsafe_blocks)]`，把当前“`unsafe` 只能留在少数可审计 FFI 边界”的约束直接固化到代码层。
* 调整 `python tools/run.py cli test` 与 host verify 的 Rust CLI 阶段：现在会先执行 `cargo clippy -- -W clippy::undocumented_unsafe_blocks`，再执行 `cargo test`，让新增 `unsafe` 缺少 `SAFETY` 注释时能在本地与 CI 真实暴露。

### 修复 (Fixed)
* 修复 Flash hostile signal 相关 host smoke 断言仍停留在旧 `450 / 900 Hz` 口径的问题；CLI 所依赖的当前 flash hostile signal 实现和设计已经改为 jitter band，名义 carrier 口径为 `559 / 1118 Hz`，相应 host verify 测试已同步到现行实现，避免 CI 因过期预期误报失败。
