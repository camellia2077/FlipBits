## 2026-05-24

### 新增功能 (Added)
* 新增 Web 生成中进度显示，浏览器侧现在通过 wasm bridge 消费 `libs` 的 encode operation snapshot / work-plan，展示 preparing、rendering、postprocessing、finalizing 等阶段和整体百分比，而不是只等同步编码结束后一次性出结果。
* 新增 WebAssembly operation 生命周期接口，Web worker 可以 `begin -> pump -> poll -> take result -> abort`，并读取 state、phase、work units、payload byte count、segment count 和 terminal code 等生成状态数据。
* 新增首页 mode 卡片和更明确的 hero / preview / about 分区，用户可以直接点击 `mini / flash / pro / ultra` 卡片切换 mode，并在同一屏完成输入、生成、预览和下载 WAV。

### 技术改进/重构 (Changed/Refactor)
* 重构 Web 编码链路，切到 worker 驱动的 operation pump 循环。前端现在通过 `ENCODE_OPERATION_STATES` 判断 queued/running/succeeded/failed/cancelled。
* 调整 wasm bridge 导出符号，补齐 operation progress、work-plan、错误码文案和 abort cleanup 入口，让 Web 与 Android 一样消费 `libs` 的生成进度事实来源。
* 调整页面结构和样式，把生成表单、结果预览、进度条、播放器和下载入口组织成更清晰的 generator / preview layout，并把项目介绍下移，避免用户打开页面时先读说明而不是先生成音频。
* 调整 i18n 文案，补充 hero copy、mode card 文案和进度显示格式，保持多语言页面在新布局和进度条下仍有完整文本。
* 精简 `apps/audio_web/README.md`，把实现细节和命令口径收口到根目录 `docs/`，并新增 Web agent 入口说明；本地静态预览服务器也改为 no-cache 响应，减少调试时旧 JS/CSS 被浏览器缓存的问题。

### 修复 (Fixed)
* 修复 Web 生成进度只能依赖旧同步编码回调、难以和 Android 共享同一套进度语义的问题。现在 Web progress 使用 operation snapshot/work-plan，后续 libs 进度字段变化时可以按同一契约同步。
* 修复生成失败或中断时 worker 侧 operation 可能残留的问题，错误路径会尽力调用 abort cleanup，并把 terminal code 映射成可读错误信息。
* 修复页面布局在加入进度和结果预览后主操作区不够聚焦的问题。现在生成入口和结果预览并列呈现，下载按钮只在有结果后启用。
