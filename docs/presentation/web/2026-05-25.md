## 2026-05-25

### 新增功能 (Added)
* 新增浏览器语言自动选择：Web demo 会优先跟随浏览器 locale，无法匹配时回退到英文，同时保留页面内手动语言切换。
* 新增 Web 端 Chrome/WebKit Playwright 冒烟测试，覆盖页面渲染、模式切换、样本文本、浏览器语言、样本长度胶囊和主工作流布局。
* 新增页面 favicon，浏览器标签页、收藏和关闭页展示 `flipbits-icon.svg`，不再使用默认图标。

### 技术改进/重构 (Changed/Refactor)
* 调整 Web 主工作台交互，把 `Mode` 下拉选择移除，统一使用 `Mode Overview` 四张模式卡片作为唯一 mode 选择入口；生成选项、输入文本和 Preview 收敛到同一个工作流区域，减少模式选择与生成结果之间的割裂。
* 调整样本文本控制，把 `Sample Length` 从下拉框改成互斥胶囊，并把按钮文案从 `Random Sample` 改为 `Random Text`，同步多语言文案。
* 调整首页引导文案，删除独立 `Start Here` 卡片，把“先选 mode、输入文本并生成音频”的提示融入工作台顶部；同时明确 Web demo 只负责文本输入和音频生成，可视化体验需要下载 Android APK。
* 调整 Web 多语言资源，补齐 `Mode Overview` 和四个模式说明的本地化文案；`mini` 模式说明从 `Morse-like` 收敛为 `Morse code`，并保留 `bit`、`16-FSK` 等技术术语不翻译。
* 调整 Repo 和 Android APK 跳转入口，在页面顶部和 `About FlipBits` 区域都提供对应链接，让用户可以直接进入仓库或下载 APK。
* 迁移 `apps/audio_web/tools` 下的 Web 构建、样本文本导出、Pages 准备和本地预览脚本到根目录 `tools` 工具体系，Web 测试也通过 `python tools/run.py web test` 统一编排。

### 修复 (Fixed)
* 修复 Web 页面在多语言切换后 `Mode Overview` 仍保留英文标题和旧 mini 说明的问题，现在模式卡片文案会随语言切换同步更新。
* 修复主工作流布局中输入标题、输入提示和样本文本工具对齐不一致的问题，让 `Input Text`、提示、样本长度、emoji 和随机文本按钮在视觉上归到同一组输入控制。
