## 2026-04-30

### 新增命令
* 新增 `python tools/run.py android kotlin-policy`，用于扫描 Android Kotlin 中直接按 Flash wire int 分支的代码，并提示改用 `FlashVoicingStyleOption` 语义 helper 或边界层常量。
* 新增 `python tools/run.py android strings-add`，封装 Android translation `add-key` 脚手架，可一次向英文基线和所有已有 `values-*` 对应 XML 文件补同一个 string key。
* 新增 `python tools/run.py verify review-fixes --build-dir build/dev`，作为 review 修复后的较短验证入口，串联 static policy、translation key alignment、`audio_api` 测试、Android `ktlint-check` 和 `assemble-debug`。

### 工作流调整
* 更新 `.agent/AGENTS.md` 与 `apps/audio_android/AGENTS.md`，把 `review-fixes`、`kotlin-policy` 和 `strings-add` 写入 agent 常用路径，减少后续处理 Android 文案或小批量 review 修复时的命令摸索。
* 更新 `docs/README.md` 与 `docs/design/android/android-translation-workflow.md`，补充“只改 Android UI 文案”和“只改 core follow / visual timeline”时的短阅读路径，并推荐新增文案 key 时优先走脚手架。
* 移除 `libs/AGENTS.md` 中不存在的旧 legacy precondition 文档引用，避免 agent 按失效路径继续查文档。

### 校验与策略
* 将 Android Kotlin policy 接入 `verify` static check group，作为通用 ktlint/detekt 之外的项目特定轻量检查，重点防止 Flash wire value magic int 在 UI 层扩散。
* 改进 `python tools/run.py android ktlint-check` 失败提示；当 ktlint 报错时会直接提示可先运行 `python tools/run.py android ktlint-format`，再重新执行 check。
* 为 Android translation `add-key` 脚手架补单测，覆盖新增 key 写入、重复运行跳过已有 key、缺失英文基线文件等行为。

### 重构
* 扩展 `tools/scripts/loc` 职责混杂扫描输出，在 JSON 和控制台中新增 `next_action` 字段，使 `scan_kt.json` / `scan_cpp.json` 不只给出风险和 suggestion，也给出更适合 agent 继续执行的第一步动作。
