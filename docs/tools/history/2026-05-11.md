## 2026-05-11

### 新增命令
* 新增 `python tools/run.py android-debug ...` 正式命令组，覆盖 adb/logcat 的设备准备、Flash/Mini/EncodeProgress 采集、summary 过滤和 crash summary，替代原先分散在 `tools/scripts/android/` 下的调试脚本入口。
* 新增 `python tools/run.py android-translate ...` 正式命令组，用于 Android XML 翻译提示词生成、资源检查、lint/autofix、key alignment 和批量替换。

### 工作流调整
* 将 Android debug 采集流程收敛到 `tools/repo_tooling/android_debug/`，旧口径是直接运行 `tools/scripts/android/*` 脚本，新口径统一通过 `tools/run.py android-debug` 下钻发现和执行。
* 将 Android translation 工具从 `tools/scripts/android/translate/` 提升到 `tools/repo_tooling/android_translate/`，旧入口 `tools/scripts/android/translate/run.py` 保留为兼容 shim，推荐入口改为 `python tools/run.py android-translate <subcommand>`。
* 更新 `python tools/run.py android strings-add` 与 `verify review-fixes`，内部改用新的 Android translation runner，避免正式工具链继续依赖 scripts 路径。
* 更新 `.agent/AGENTS.md`、`tools/README.md` 和 Android 翻译工作流文档，把 `android-debug` / `android-translate` 纳入常用工具入口和命令发现路径。

### 校验与策略
* 为 Android debug 与 Android translation 工具补充 pytest 覆盖，使 log summary、采集命令构造、翻译 lint、replacement、key alignment 等逻辑可以在没有真实手机的环境下回归。
* 调整 Android Gradle translation lint/key-alignment 入口，改用 `tools/repo_tooling/android_translate/run.py` 和新的 lint baseline 路径。

### 修复
* 修复 Android translation 工具迁移后的仓库根目录推导问题，默认 `res` 路径改为按 `tools/run.py` 与 `apps/` 标记向上定位，避免目录上移后解析到错误的 `C:\code` 根。
* 修复 `python tools/run.py message prep --history docs/tools/history/YYYY-MM-DD.md` 无法读取 tools history 的问题，现在可将 tools history 的 `新增命令`、`工作流调整`、`校验与策略`、`重构`、`修复` 映射为 commit message 草稿 sections。
