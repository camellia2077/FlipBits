## 2026-06-03

### 新增命令
* 新增 `python tools/run.py artifact flash-speed-analysis`，用于在 host 上生成 Flash 各 style 的源 WAV，使用 Android `SpeedAdjustedPcmRenderer` 的 host mirror 渲染指定播放倍速 PCM，并输出 spectral / amplitude 对比指标。
* 默认分析全部六个 Flash style 与 `0.1x`，产物写入 `build/test-artifacts/flash-speed-analysis/` 下的 `summary.md`、`metrics.json`、`metrics.csv` 和源 WAV。
* `python tools/run.py android-debug capture-playback-speed` 和 `playback-speed-summary` 现在会捕获 / 汇总 Android `PlaybackSpeedMemory` logcat 行，用于 speed-adjusted playback 的内存诊断。

### 工作流调整
* 排查 Flash speed 的 style-specific pitch / timbre 报告时，优先跑 `flash-speed-analysis`，先确认 app 内部源 PCM 与 speed-adjusted PCM 的客观变化，再决定是否改 renderer 或检查 Android 播放输出链路。
* 排查播放内存问题时，先采集 `PlaybackSpeedMemory` 指标，再决定是否优化。指标覆盖 renderer 类型、模式、speed、streaming/file-backed、source/rendered samples、估算 PCM bytes、Java/native heap delta、source load time 或 render time。若 summary 没有 `Memory Rows`，说明 capture 窗口内没有进入 speed-adjusted render path。
