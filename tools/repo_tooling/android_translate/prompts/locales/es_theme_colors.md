# Spanish Theme And Color Translation Rules

Use this note when the Spanish task covers theme titles, palette names, dual-tone descriptions, accessibility labels, or custom color import/export UI.

## Theme Titles

- Write theme titles as natural neutral-Spanish product names.
- Avoid stiff word-for-word calques when a smoother Spanish title works.
- Keep titles compact enough for a settings picker.
- Use capitalization consistent with normal Spanish title-style UI labels rather than decorative English headline casing.

## Dual-Tone Descriptions

- Keep dual-tone descriptions short and color-led.
- Prefer compact color-list phrasing such as `Color A y color B.` or `Color A, color B y color C.`
- Do not turn the description line into atmospheric prose, lore copy, or marketing-style narration.
- Avoid repeating `paleta bitono`, `aspecto` or similar framing words when the theme context is already visible in UI.

## Accessibility Labels

- Prefer a clear pattern such as `Tema Nombre, color ...`
- Keep color names explicit and easy to scan in screen-reader output.
- Avoid unnecessary flourish; accessibility text should describe the visible palette directly.

## Material Palette Names

- Use natural neutral-Spanish color names when they sound normal in a palette picker.
- Keep palette labels compact and product-like.
- Loanwords are acceptable when they are already standard color vocabulary in Spanish UI.
- Avoid mixing very literal phrases with highly localized color words without a clear pattern.

## Product-Term Consistency

- Keep `Dual-Tone` as a stable product term when the UI is referring to the product/system name.
- Use `bitono` only when the text is clearly describing a color style in ordinary Spanish rather than naming the product option.
- Keep `Material` unchanged as the design-system family name.
