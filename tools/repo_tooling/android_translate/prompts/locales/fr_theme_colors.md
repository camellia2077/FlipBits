# French Theme And Color Translation Rules

Use this note when the French task covers theme titles, palette names, dual-tone descriptions, accessibility labels, or custom color import/export UI.

## Theme Titles

- Write theme titles as natural French product names.
- Avoid stiff word-for-word calques when a smoother French title works.
- Keep titles compact enough for a settings picker.
- Use normal French title casing for UI labels rather than decorative English headline casing.

## Dual-Tone Descriptions

- Keep dual-tone descriptions short and color-led.
- Prefer compact color-list phrasing such as `Couleur A et couleur B.` or `Couleur A, couleur B et couleur C.`
- Do not turn the description line into atmosphere prose, lore copy, or marketing narration.
- Avoid repeating `palette Dual-Tone`, `allure`, or similar framing words when the theme context is already obvious in UI.

## Accessibility Labels

- Prefer a clear pattern such as `Thème Nom, couleur ...`
- Keep color names explicit and easy to scan in screen-reader output.
- Accessibility text should describe the visible palette directly, without decorative expansion.

## Material Palette Names

- Use natural French color names when they sound normal in a palette picker.
- Keep palette labels compact and product-like.
- Loanwords are acceptable when they are already standard French color vocabulary.
- Avoid mixing very literal phrases with overly editorial wording without a clear pattern.

## Product-Term Consistency

- Keep `Dual-Tone` as a stable product term when the UI is referring to the product/system name.
- Use `bicolore` only when the string is clearly describing an ordinary two-color style rather than naming the product option.
- Keep `Material` unchanged as the design-system family name.
