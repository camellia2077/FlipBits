# High Gothic Dog Latin Theme And Color Rules

Use this note when the High Gothic (`values-la`) task covers theme titles, palette names, dual-tone descriptions, accessibility labels, or custom color import/export UI.

## Theme Titles

- Write theme titles as ceremonial product names in High Gothic / Dog Latin style.
- Favor compact, sonorous names over textbook-Latin literalism.
- Keep titles short enough for a settings picker.
- Selective English carry-over is acceptable only when it already reads like a house term.

## Dual-Tone Descriptions

- Keep dual-tone descriptions short and color-led.
- Prefer compact lists such as `Color A, color B.` or `Color A, color B, et color C.`
- Do not pad the line with lore narration when the UI already shows the theme context.
- Avoid repeating `paletta Dual-tone`, `sigillum`, or similar framing words in every description.

## Accessibility Labels

- Prefer a clear pattern such as `Thema Nomen, color ...`
- Keep visible colors explicit and easy to scan in screen-reader output.
- Accessibility text should describe the palette directly without decorative expansion.

## Material Palette Names

- Keep `Material` unchanged as the design-system family name.
- Palette labels may be Latinized, pseudo-Latin, or stable loan forms if they read like concise preset names.
- Prefer short product-name rhythm over strict lexical consistency.

## Product-Term Consistency

- Keep `Dual-tone` as the stable product/system term.
- Use `custom` only when it helps preserve recognizability in import/export or config wording.
- Reuse existing house forms where nearby strings establish them, especially `High Gothic`, `Duo-tone`, `Material`, and `config`.
