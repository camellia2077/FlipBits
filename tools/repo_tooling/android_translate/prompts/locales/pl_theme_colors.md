# Polish Theme And Color Translation Rules

Use this note when the Polish task covers theme titles, palette names, dual-tone descriptions, accessibility labels, or custom color import/export UI.

## Theme Titles

- Write theme titles as natural Polish product names.
- Avoid stiff word-for-word calques when a smoother Polish title works.
- Keep titles compact enough for a settings picker.
- Use normal Polish UI capitalization rather than decorative English headline casing.

## Dual-Tone Descriptions

- Keep dual-tone descriptions short and color-led.
- Prefer compact color-list phrasing such as `Kolor A i kolor B.` or `Kolor A, kolor B i kolor C.`
- Do not turn the description line into lore copy, atmosphere prose, or marketing narration.
- Avoid repeating `paleta dual-tone`, `wygląd`, or similar framing words when the theme context is already obvious in UI.

## Accessibility Labels

- Prefer a clear pattern such as `Motyw Nazwa, kolor ...`
- Keep color names explicit and easy to scan in screen-reader output.
- Accessibility text should describe the visible palette directly, without decorative expansion.

## Material Palette Names

- Keep `Material` unchanged as the design-system family name.
- Use natural Polish color names when they sound normal in a palette picker.
- Keep palette labels compact and product-like.

## Product-Term Consistency

- Keep `dual-tone` as the stable product/system term.
- Prefer `własny` for user-created themes and palettes.
- In import/export UI, prefer short product wording over literal expansion.
