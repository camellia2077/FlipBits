# Brazilian Portuguese Theme And Color Translation Rules

Use this note when the Brazilian Portuguese task covers theme titles, palette names, dual-tone descriptions, accessibility labels, or custom color import/export UI.

## Theme Titles

- Write theme titles as natural Brazilian Portuguese product names.
- Avoid stiff word-for-word calques when a smoother Brazilian title works.
- Keep titles compact enough for a settings picker.
- Use normal Brazilian Portuguese UI capitalization rather than decorative English headline casing.

## Dual-Tone Descriptions

- Keep dual-tone descriptions short and color-led.
- Prefer compact color-list phrasing such as `Cor A e cor B.` or `Cor A, cor B e cor C.`
- Do not turn the description line into lore copy, atmosphere prose, or marketing narration.
- Avoid repeating `paleta dual-tone`, `visual`, or similar framing words when the theme context is already obvious in UI.

## Accessibility Labels

- Prefer a clear pattern such as `Tema Nome, cor ...`
- Keep color names explicit and easy to scan in screen-reader output.
- Accessibility text should describe the visible palette directly, without decorative expansion.

## Material Palette Names

- Keep `Material` unchanged as the design-system family name.
- Use natural Brazilian Portuguese color names when they sound normal in a palette picker.
- Keep palette labels compact and product-like.

## Product-Term Consistency

- Keep `dual-tone` as the stable product/system term.
- Prefer `personalizado` for user-created themes and palettes.
- In import/export UI, prefer short Brazilian product wording over literal expansion.
